## 1. Guest chat (no login)

**Use this when:**

- Anyone should be able to start chatting instantly  
- You do not have (or do not want) your own login form yet  

### Copy this into `<head>`

```html
<script defer src="https://cdn.jsdelivr.net/npm/@cometchat/chat-embed@1.x.x/dist/main.js"></script>
```

### Copy this into `<body>`

```html lines highlight={9-10, 12, 16-18, 23-25}
<!-- Where the chat widget should appear -->
<div id="cometChatMount"></div>

<script type="module">
  import { CometChatAuth } from "https://cdn.example.com/cometchat/cometchat-auth.js";

  //  EDIT ONLY THE TEXT INSIDE QUOTES
  const COMETCHAT_WIDGET_CONFIG = {
    appID: "YOUR_APP_ID",
    region: "YOUR_APP_REGION",   // e.g. "us", "eu", "in"
    mode: "guest",               // keep this as "guest"
    authKey: "YOUR_AUTH_KEY",

    // Optional: how the guest appears in chat
    user: {
      name: "Guest User",
      avatar: "",                // e.g. "https://example.com/guest.png"
      link: ""                   // e.g. "https://example.com/profile"
    },

    // Widget placement + size
    mount: "#cometChatMount",
    width: "450px",
    height: "80vh",
    isDocked: true               // true = docked widget (recommended)
  };

  CometChatAuth.start(COMETCHAT_WIDGET_CONFIG);
</script>
```

### Update these values

- `appID`, `region` – copy them from **Dashboard → Settings → App Details**.  
- `authKey` – use the **Guest Auth Key** from **Dashboard → Settings → API & Auth Keys**.  
- `user.name`, `user.avatar`, `user.link` – optional display info for the visitor.  
- `width`, `height`, `isDocked` – tweak how the widget looks on the page.

## 2. Log in with UID (create user automatically)

**Use this when:**

- Every person already has an ID inside your product (email, username, order number, etc.)  
- You want that ID to drive their chat identity automatically  

### Copy this into `<head>`

```html
<script defer src="https://cdn.jsdelivr.net/npm/@cometchat/chat-embed@1.x.x/dist/main.js"></script>
```

### Copy this into `<body>`

```html lines highlight={9-10, 12, 15, 19-21, 26-28}
<!-- Where the chat widget should appear -->
<div id="cometChatMount"></div>

<script type="module">
  import { CometChatAuth } from "https://cdn.example.com/cometchat/cometchat-auth.js";

  //  EDIT ONLY THE TEXT INSIDE QUOTES
  const COMETCHAT_WIDGET_CONFIG = {
    appID: "YOUR_APP_ID",
    region: "YOUR_APP_REGION",      // e.g. "us", "eu", "in"
    mode: "uid",                    // keep this as "uid"
    authKey: "YOUR_AUTH_KEY",

    // IMPORTANT: this must be unique per user
    uid: "UNIQUE_USER_ID",          // e.g. "user_123", "customer_42"

    // Optional: user profile shown in chat
    user: {
      name: "User Name",            // e.g. "Alex Johnson"
      avatar: "https://example.com/uid.png",
      link: "https://example.com/profile/uid"
    },

    // Widget placement + size
    mount: "#cometChatMount",
    width: "450px",
    height: "80vh",
    isDocked: true
  };

  CometChatAuth.start(COMETCHAT_WIDGET_CONFIG);
</script>
```

### Update these values

- `appID`, `region`, `authKey` – same as the guest flow.  
- `uid` – pass the unique ID from the page (for example `customer.id`). CometChat will create the user the first time it sees that ID.  
- `user.name`, `user.avatar`, `user.link` – optional profile info your visitors already have.  
- Keep `mode: "uid"` so CometChat knows you are logging people in with IDs.

## 3. Log in with your backend (auth token login)

**Use this when:**

- People sign in to your product through your own backend  
- You can create a CometChat auth token on the server after they sign in  

### Copy this into `<head>`

```html
<script defer src="https://cdn.jsdelivr.net/npm/@cometchat/chat-embed@1.x.x/dist/main.js"></script>
```

### Copy this into `<body>`

```html lines highlight={9-10, 14, 18-19}
<!-- Where the chat widget should appear -->
<div id="cometChatMount"></div>

<script type="module">
  import { CometChatAuth } from "https://cdn.example.com/cometchat/cometchat-auth.js";

  //  EDIT ONLY THE TEXT INSIDE QUOTES
  const COMETCHAT_WIDGET_CONFIG = {
    appID: "YOUR_APP_ID",
    region: "YOUR_APP_REGION",       // e.g. "us", "eu", "in"
    mode: "authToken",               // keep this as "authToken"

    // Generated BY YOUR BACKEND after user login
    authToken: "USER_AUTH_TOKEN",

    // Widget placement + size
    mount: "#cometChatMount",
    width: "450px",
    height: "80vh",
    isDocked: true
  };

  CometChatAuth.start(COMETCHAT_WIDGET_CONFIG);
</script>
```

### Update these values

- `appID`, `region` – from **Dashboard → Settings → App Details**.  
- Keep `mode: "authToken"`.  
- `authToken` – **generate this on your server** after the user logs in:  
  1. Your server calls the CometChat Auth Token API (or server SDK) using the REST API key.  
  2. CometChat returns an auth token tied to that user.  
  3. Send the token to your page (template variable, AJAX call, etc.) and place it in `authToken`.  
- The rest of the config (`mount`, `width`, `height`, `isDocked`) controls layout just like the other flows.
