## Summary

User Presence in the CometChat JavaScript SDK enables real-time tracking of user online/offline status. This feature allows your application to display whether users are currently available to chat, when they were last active, and receive instant notifications when users come online or go offline.

## When to Use This

- When you need to show online/offline indicators next to user avatars
- When displaying "last seen" timestamps for offline users
- When building presence-aware features like "available to chat" indicators
- When you want to notify users when their contacts come online
- When implementing typing indicators or activity status in chat interfaces

## Prerequisites

- **SDK**: `@cometchat/chat-sdk-javascript` v4.x
- **State**: User must be logged in via `CometChat.login()`
- **Dashboard**: Ensure your CometChat app is created in the [CometChat Dashboard](https://app.cometchat.com)
- **AppSettings**: Presence subscription must be configured during SDK initialization

## Minimal Example

<Tabs>
<Tab title="JavaScript">
```javascript
// Configure presence subscription during init
let appSettings = new CometChat.AppSettingsBuilder()
  .subscribePresenceForAllUsers()
  .setRegion(REGION)
  .build();

CometChat.init(APP_ID, appSettings).then(
  () => {
    console.log("Initialization completed successfully");
    
    // Add presence listener after login
    let listenerId = "UNIQUE_LISTENER_ID";
    CometChat.addUserListener(
      listenerId,
      new CometChat.UserListener({
        onUserOnline: (onlineUser) => {
          console.log("User is online:", onlineUser);
        },
        onUserOffline: (offlineUser) => {
          console.log("User is offline:", offlineUser);
        }
      })
    );
  },
  (error) => {
    console.log("Initialization failed with error:", error);
  }
);
```

## Step-by-Step

### Step 1: Configure Presence Subscription

Before receiving presence updates, you must configure which users' presence you want to subscribe to. This is done during SDK initialization using the `AppSettingsBuilder`.

The `AppSettingsBuilder` provides three methods for presence subscription:

| Method | Description |
| ------ | ----------- |
| `subscribePresenceForAllUsers()` | Receive presence updates for all users in the app |
| `subscribePresenceForRoles(roles)` | Receive presence updates only for users with specified roles |
| `subscribePresenceForFriends()` | Receive presence updates only for the logged-in user's friends |

<Note>
If none of these methods are called during initialization, no presence updates will be sent to the logged-in user.
</Note>

### Step 2: Register User Listener

To receive real-time presence updates, register a `UserListener` using the `addUserListener()` method. The listener provides callbacks for when users come online or go offline.

<Tabs>
<Tab title="JavaScript">
```javascript
let listenerId = "UNIQUE_LISTENER_ID";

CometChat.addUserListener(
  listenerId,
  new CometChat.UserListener({
    onUserOnline: (onlineUser) => {
      console.log("User is online:", onlineUser);
      // Update UI to show online status
    },
    onUserOffline: (offlineUser) => {
      console.log("User is offline:", offlineUser);
      // Update UI to show offline status
    }
  })
);
```
</Tab>

### Step 4: Check Presence from User List

When fetching users, you can check their presence status directly from the User object without relying on real-time events.

<Tabs>
<Tab title="JavaScript">
```javascript
let usersRequest = new CometChat.UsersRequestBuilder()
  .setLimit(30)
  .build();

usersRequest.fetchNext().then(
  (userList) => {
    userList.forEach((user) => {
      let status = user.getStatus(); // "online" or "offline"
      let lastActiveAt = user.getLastActiveAt(); // Timestamp (if offline)
      
      console.log(`${user.getName()} is ${status}`);
      if (status === "offline" && lastActiveAt) {
        console.log(`Last seen: ${new Date(lastActiveAt * 1000)}`);
      }
    });
  },
  (error) => {
    console.log("User list fetching failed:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">

### Event Ordering

- `onUserOnline` is triggered when a user establishes a connection to CometChat
- `onUserOffline` is triggered when a user disconnects or their session times out
- Events are delivered in real-time but may have slight delays based on network conditions
- If a user rapidly connects and disconnects, you may receive multiple events in quick succession

## API Details

### Method Signatures

```javascript
// AppSettingsBuilder methods
AppSettingsBuilder.subscribePresenceForAllUsers(): AppSettingsBuilder
AppSettingsBuilder.subscribePresenceForRoles(roles: string[]): AppSettingsBuilder
AppSettingsBuilder.subscribePresenceForFriends(): AppSettingsBuilder

// User Listener methods
CometChat.addUserListener(listenerId: string, listener: UserListener): void
CometChat.removeUserListener(listenerId: string): void

// User object methods
User.getStatus(): string
User.getLastActiveAt(): number
```

### Parameters

#### addUserListener

| Parameter  | Type         | Required | Description                                      |
| ---------- | ------------ | -------- | ------------------------------------------------ |
| listenerId | string       | Yes      | Unique identifier for the listener               |
| listener   | UserListener | Yes      | UserListener instance with callback methods      |

#### subscribePresenceForRoles

| Parameter | Type     | Required | Description                                      |
| --------- | -------- | -------- | ------------------------------------------------ |
| roles     | string[] | Yes      | Array of role names to subscribe to              |

### Return Values

#### User.getStatus()

Returns a `string` with one of two values:
- `"online"` - User is currently connected and available
- `"offline"` - User is not currently connected

#### User.getLastActiveAt()

Returns a `number` representing the Unix timestamp (in seconds) of when the user was last online. Returns `0` or `undefined` if the user is currently online or has never been online.

### Errors

| Error Code           | Description                              | Resolution                                       |
| -------------------- | ---------------------------------------- | ------------------------------------------------ |
| ERR_NOT_LOGGED_IN    | User is not logged in                    | Ensure user is logged in via CometChat.login()   |
| ERR_INVALID_LISTENER | Invalid listener object provided         | Ensure UserListener is properly instantiated     |

## Edge Cases & Pitfalls

<Warning>
**Presence Subscription Required**: If you don't configure presence subscription during `CometChat.init()`, you will not receive any presence updates. This is a common oversight—always call one of the `subscribePresenceFor*` methods in your AppSettingsBuilder.
</Warning>

<Warning>
**Duplicate Listener IDs**: Using the same `listenerId` for multiple listeners will overwrite the previous listener. Always use unique listener IDs, especially when registering listeners in multiple components. Consider using component-specific prefixes or timestamps.
</Warning>

<Warning>
**Memory Leaks from Unremoved Listeners**: Failing to remove listeners when components unmount or views change can cause memory leaks and duplicate event handling. Always call `removeUserListener()` in cleanup code (e.g., React's `useEffect` cleanup, Vue's `onUnmounted`, or Angular's `ngOnDestroy`).
</Warning>

<Warning>
**lastActiveAt Timestamp Format**: The `getLastActiveAt()` method returns a Unix timestamp in seconds, not milliseconds. When converting to a JavaScript Date, multiply by 1000: `new Date(lastActiveAt * 1000)`.
</Warning>

## Performance & Scale Notes

- Presence updates are delivered via WebSocket for real-time performance
- Using `subscribePresenceForAllUsers()` in apps with many users may generate significant event traffic—consider using role-based or friends-only subscriptions for better performance
- The `status` and `lastActiveAt` fields are included in User objects fetched via `UsersRequest`, so you can display presence without additional API calls
- Presence status is cached on the server; there may be a brief delay (typically under 30 seconds) between a user disconnecting and the offline event being triggered

## Security Notes

<Note>
Presence information is available to all users based on the subscription configuration. There is no per-user privacy control for presence—if a user is subscribed to presence updates, they will see the online/offline status of applicable users.
</Note>

<Note>
For applications requiring presence privacy features, consider implementing application-level logic to filter which users' presence is displayed, or use role-based subscriptions to limit visibility.
</Note>
