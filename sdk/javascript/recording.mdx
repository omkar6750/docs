## Summary

Call recording enables you to capture audio and video from voice and video calls for later playback, compliance, or training purposes. This guide covers configuring recording settings, starting and stopping recordings programmatically, handling recording events via listeners, and accessing recorded files from the CometChat Dashboard.

## When to Use This

- Record calls for compliance, legal, or regulatory requirements
- Capture training sessions or customer support calls for quality assurance
- Enable users to record important meetings or conversations for later review
- Implement automatic recording that starts when a call begins
- Build custom recording controls with your own UI buttons

## Prerequisites

1. **Chat SDK initialized**: Ensure `CometChat.init()` has completed successfully. See [Setup SDK](/sdk/javascript/setup-sdk).
2. **Calls SDK initialized**: Ensure `CometChatCalls.init()` has completed. See [Calling Setup](/sdk/javascript/calling-setup).
3. **User logged in**: A user must be logged in via `CometChat.login()`. See [Authentication](/sdk/javascript/authentication-overview).
4. **Active call session**: Recording can only be started during an active call session. See [Call Session](/sdk/javascript/direct-call) or [Ringing](/sdk/javascript/default-call).
5. **Recording enabled**: Ensure call recording is enabled in your [CometChat Dashboard](https://app.cometchat.com) under Calls settings.

## Minimal Example

<Tabs>
<Tab title="JavaScript">
```javascript
const callListener = new CometChatCalls.OngoingCallListener({
  onRecordingStarted: (event) => {
    console.log("Recording started by:", event.user);
  },
  onRecordingStopped: (event) => {
    console.log("Recording stopped by:", event.user);
  }
});

const callSettings = new CometChatCalls.CallSettingsBuilder()
  .enableDefaultLayout(true)
  .setIsAudioOnlyCall(false)
  .showRecordingButton(true)
  .setCallListener(callListener)
  .build();

const htmlElement = document.getElementById("call-container");
CometChatCalls.startSession(callToken, callSettings, htmlElement);
```

## Step-by-Step

### Step 1: Configure Call Settings for Recording

Before starting a call session, configure the `CallSettings` to enable recording features. You can show the recording button in the default UI or enable automatic recording when the call starts.

<Tabs>
<Tab title="JavaScript">
```javascript
const callSettings = new CometChatCalls.CallSettingsBuilder()
  .enableDefaultLayout(true)
  .setIsAudioOnlyCall(false)
  .showRecordingButton(true)                    // Show recording button in UI
  .startRecordingOnCallStart(false)             // Don't auto-record
  .setCallListener(callListener)
  .build();
```
</Tab>
<Tab title="TypeScript">
```typescript
const callSettings = new CometChatCalls.CallSettingsBuilder()
  .enableDefaultLayout(true)
  .setIsAudioOnlyCall(false)
  .showRecordingButton(true)                    // Show recording button in UI
  .startRecordingOnCallStart(false)             // Don't auto-record
  .setCallListener(callListener)
  .build();
```
</Tab>
</Tabs>

### Step 2: Add Recording Event Listeners

Register listeners to receive notifications when recording starts or stops. This allows you to update your UI or perform actions based on recording state changes.

<Tabs>
<Tab title="JavaScript">
```javascript
const listenerId = "UNIQUE_LISTENER_ID";

const callListener = new CometChatCalls.OngoingCallListener({
  onRecordingStarted: (event) => {
    console.log("Recording started by:", event.user);
    // Update UI to show recording indicator
  },
  onRecordingStopped: (event) => {
    console.log("Recording stopped by:", event.user);
    // Update UI to hide recording indicator
  },
  onUserJoined: (user) => {
    console.log("User joined:", user);
  },
  onCallEnded: () => {
    console.log("Call ended");
  }
});
```
</Tab>

## API Details

### Recording Settings

Configure recording behavior using the `CallSettingsBuilder` methods:

| Method | Type | Default | Description |
| ------ | ---- | ------- | ----------- |
| `showRecordingButton(show)` | `boolean` | `false` | Shows or hides the recording button in the default call UI layout. |
| `startRecordingOnCallStart(start)` | `boolean` | `false` | When `true`, recording starts automatically when the call session begins. |

### Recording Methods

| Method | Description |
| ------ | ----------- |
| `CometChatCalls.startRecording()` | Starts recording the current call session. Can only be called during an active session. |
| `CometChatCalls.stopRecording()` | Stops the current recording. The recording file will be processed and made available in the Dashboard. |

### Recording Events

| Event | Description |
| ----- | ----------- |
| `onRecordingStarted(event)` | Invoked when recording starts. The `event.user` contains information about who started the recording. |
| `onRecordingStopped(event)` | Invoked when recording stops. The `event.user` contains information about who stopped the recording. |

### Return Values

The `startRecording()` and `stopRecording()` methods do not return values. Use the event listeners to track recording state changes.

### Errors

| Error | Description | Resolution |
| ----- | ----------- | ---------- |
| Recording not enabled | Recording feature is not enabled for your app | Enable recording in the CometChat Dashboard under Calls settings |
| No active session | Attempted to start/stop recording without an active call | Ensure a call session is active before calling recording methods |
| Recording already in progress | Attempted to start recording when already recording | Check recording state before calling `startRecording()` |

## Events & Real-time Behavior

### Registering Recording Listeners

Add recording event listeners as part of your `OngoingCallListener` configuration:

<Tabs>
<Tab title="JavaScript">
```javascript
const listenerId = "UNIQUE_LISTENER_ID";

const callListener = new CometChatCalls.OngoingCallListener({
  onRecordingStarted: (event) => {
    console.log("Recording started by:", event.user);
    // Show recording indicator in your UI
    showRecordingIndicator(true);
  },
  onRecordingStopped: (event) => {
    console.log("Recording stopped by:", event.user);
    // Hide recording indicator
    showRecordingIndicator(false);
  }
});

const callSettings = new CometChatCalls.CallSettingsBuilder()
  .setCallListener(callListener)
  .build();
```
</Tab>
<Tab title="TypeScript">

    // Show recording indicator in your UI
    showRecordingIndicator(true);
  },
  onRecordingStopped: (event: { user: any }) => {
    console.log("Recording stopped by:", event.user);
    // Hide recording indicator

### Removing Listeners

Listeners are automatically cleaned up when you call `CometChatCalls.endSession()`. If you registered additional listeners via `addCallEventListener()`, remove them explicitly:

<Tabs>
<Tab title="JavaScript">
```javascript
// Remove listener when done
CometChatCalls.removeCallEventListener(listenerId);
```
</Tab>
<Tab title="TypeScript">
```typescript
// Remove listener when done
CometChatCalls.removeCallEventListener(listenerId);
```
</Tab>
</Tabs>

### Event Ordering

1. When a user starts recording, `onRecordingStarted` fires for all participants
2. Recording continues until explicitly stopped or the call ends
3. When recording stops, `onRecordingStopped` fires for all participants
4. If the call ends while recording, the recording is automatically stopped and saved

## Edge Cases & Pitfalls

<Warning>
**Recording Without Active Session**: Calling `startRecording()` or `stopRecording()` without an active call session will fail silently. Always ensure a call session is active before attempting to control recording.
</Warning>

<Warning>
**Multiple Recording Starts**: Calling `startRecording()` when recording is already in progress may cause unexpected behavior. Track the recording state in your application and disable the start button when recording is active.
</Warning>

<Warning>
**Recording Not Enabled in Dashboard**: If call recording is not enabled in your CometChat Dashboard, the recording button will not appear and programmatic recording calls will fail. Verify your Dashboard settings before implementing recording features.
</Warning>

<Warning>
**Recording Indicator for All Participants**: When one participant starts recording, all participants receive the `onRecordingStarted` event. Ensure your UI shows a recording indicator to all users for transparency and compliance with recording consent laws.
</Warning>

<Warning>
**Large Recording Files**: Long calls produce large recording files. Consider implementing recording duration limits or warnings for extended recordings to manage storage costs and processing time.
</Warning>

## Performance & Scale Notes

- **Recording Processing**: Recordings are processed asynchronously after the call ends. Large recordings may take several minutes to become available in the Dashboard.
- **Storage Considerations**: Recording files are stored in CometChat's cloud storage. Monitor your usage in the Dashboard to manage storage costs.
- **Bandwidth Impact**: Recording adds minimal overhead to call bandwidth as it captures the media streams server-side.
- **Concurrent Recordings**: Each call session can have one active recording at a time. Starting a new recording while one is in progress will not create multiple recordings.

## Security Notes

<Note>
**Auth Key** is acceptable for POC/development only. For production, use **Auth Token** created server-side via the CometChat REST API.
</Note>

<Note>
**Recording Consent**: Many jurisdictions require consent from all parties before recording a call. Implement appropriate consent mechanisms and display recording indicators to comply with local laws and regulations.
</Note>

<Note>
**Recording Access**: Call recordings are accessible only through the CometChat Dashboard by authorized administrators. Implement appropriate access controls in your organization.
</Note>

## Downloading Recordings

Call recordings are available in the [CometChat Dashboard](https://app.cometchat.com) under the Calls section. To access recordings:

1. Navigate to the **Calls** section in your Dashboard
2. Find the call you want to review
3. Click the three-dot menu icon to expand options
4. Select **View Recordings** to access the recording files
