## Summary

The CometChat JavaScript SDK enables you to change the scope (role) of group members. Group admins can promote participants to moderators or admins, or demote members to lower roles. This is essential for building dynamic group hierarchies, delegating moderation responsibilities, and implementing role-based access control within groups.

## When to Use This

- When implementing role management features in group settings
- When promoting active members to moderators or admins
- When demoting members who no longer need elevated privileges
- When building admin dashboards for group management
- When implementing automated role assignment based on user activity
- When creating tiered permission systems within groups

## Prerequisites

- **SDK**: `@cometchat/chat-sdk-javascript` v4.x
- **State**: User must be logged in via `CometChat.login()`
- **Dashboard**: Ensure your CometChat app is created in the [CometChat Dashboard](https://app.cometchat.com)
- **Group**: The group must exist and the logged-in user must be an admin
- **Permissions**: Only group admins can change member scopes

## Step-by-Step

### Step 1: Understand Group Member Scopes

CometChat groups support three member scopes (roles):

| Scope       | Constant                                | Description                                      |
| ----------- | --------------------------------------- | ------------------------------------------------ |
| Participant | `CometChat.GROUP_MEMBER_SCOPE.PARTICIPANT` | Default role with basic messaging permissions    |
| Moderator   | `CometChat.GROUP_MEMBER_SCOPE.MODERATOR`   | Can kick/ban participants                        |
| Admin       | `CometChat.GROUP_MEMBER_SCOPE.ADMIN`       | Full control including promoting/demoting members |

The default scope for any new member is `participant`. Only **Admins** can change the scope of other members.

## Events & Real-time Behavior

### Registering GroupListener

To receive real-time events when a member's scope is changed, register a `GroupListener` with the `onGroupMemberScopeChanged` callback.

<Tabs>
<Tab title="JavaScript">
```javascript
let listenerId = "unique-scope-change-listener";

CometChat.addGroupListener(
  listenerId,
  new CometChat.GroupListener({
    onGroupMemberScopeChanged: (message, changedUser, newScope, oldScope, changedGroup) => {
      console.log("Member scope changed:", {
        message,
        changedUser,
        newScope,
        oldScope,
        changedGroup,
      });
    },
  })
);
```
</Tab>
<Tab title="TypeScript">
```typescript
let listenerId: string = "unique-scope-change-listener";

CometChat.addGroupListener(
  listenerId,
  new CometChat.GroupListener({
    onGroupMemberScopeChanged: (
      message: CometChat.Action,
      changedUser: CometChat.User,
      newScope: string,
      oldScope: string,
      changedGroup: CometChat.Group
    ) => {
      console.log("Member scope changed:", {
        message,
        changedUser,
        newScope,
        oldScope,
        changedGroup,
      });
    },
  })
);
```
</Tab>
</Tabs>

### Complete Listener Lifecycle Example

<Tabs>
<Tab title="JavaScript">
```javascript
// Use a unique listener ID
let listenerId = "scope-change-listener-" + Date.now();

// Add the listener
CometChat.addGroupListener(
  listenerId,
  new CometChat.GroupListener({
    onGroupMemberScopeChanged: (message, changedUser, newScope, oldScope, changedGroup) => {
      console.log(`${changedUser.getName()}'s scope changed from ${oldScope} to ${newScope} in ${changedGroup.getName()}`);
    },
  })
);

// Later, when cleaning up (e.g., component unmount)
function cleanup() {
  CometChat.removeGroupListener(listenerId);
```

### Scope Change Events in Message History

When retrieving previous messages, if a member's scope was changed in a group, the message list will contain an `Action` message. The `Action` object provides the following information:

| Property  | Description                                                |
| --------- | ---------------------------------------------------------- |
| action    | The action type: `"scopeChanged"`                          |
| actionOn  | User object of the user whose scope was changed            |
| actionBy  | User object of the user who changed the scope              |
| actionFor | Group object of the group where the scope was changed      |
| oldScope  | The original scope of the member                           |
| newScope  | The updated scope of the member                            |

## API Details

### updateGroupMemberScope()

```javascript
CometChat.updateGroupMemberScope(GUID: string, UID: string, scope: string): Promise<boolean>
```

| Parameter | Type   | Required | Description                                                                                                                                                 |
| --------- | ------ | -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| GUID      | string | Yes      | The unique identifier of the group                                                                                                                          |
| UID       | string | Yes      | The unique identifier of the member whose scope to change                                                                                                   |
| scope     | string | Yes      | The new scope: `CometChat.GROUP_MEMBER_SCOPE.ADMIN`, `CometChat.GROUP_MEMBER_SCOPE.MODERATOR`, or `CometChat.GROUP_MEMBER_SCOPE.PARTICIPANT` |

### Return Value

Returns a `Promise<boolean>` that resolves to `true` when the scope change is successful.

### Errors

| Error Code              | Description                                    | Resolution                                      |
| ----------------------- | ---------------------------------------------- | ----------------------------------------------- |
| ERR_USER_NOT_LOGGED_IN  | No user is currently logged in                 | Ensure user is logged in before performing action |
| ERR_INVALID_GUID        | The provided GUID is invalid                   | Verify the GUID format and existence            |
| ERR_GROUP_NOT_FOUND     | Group with the specified GUID not found        | Check if the group exists                       |
| ERR_NOT_A_MEMBER        | Target user is not a member of the group       | Verify the user is a group member               |
| ERR_PERMISSION_DENIED   | User doesn't have permission                   | Only admins can change member scopes            |
| ERR_USER_NOT_FOUND      | The specified UID doesn't exist                | Verify the UID exists in your CometChat app     |
| ERR_INVALID_SCOPE       | The provided scope value is invalid            | Use one of the valid scope constants            |

## Edge Cases & Pitfalls

<Warning>
**Admin-Only Permission**: Only group admins can change member scopes. Moderators and participants cannot promote or demote other members. Attempting to change scope without admin privileges will result in a permission error.
</Warning>

<Warning>
**Cannot Change Owner Scope**: The group owner's scope cannot be changed. The owner always has the highest privileges. To transfer ownership, use the `transferGroupOwnership()` method instead.
</Warning>

<Warning>
**Duplicate Listener IDs**: If you register a GroupListener with the same `listenerId` as an existing listener, the new listener will replace the old one. Use unique listener IDs (e.g., include timestamps or UUIDs) to avoid accidentally overwriting listeners.
</Warning>

<Warning>
**Self-Demotion Risk**: An admin can demote themselves to participant or moderator. Once demoted, they cannot promote themselves back. Implement UI safeguards to prevent accidental self-demotion, especially for the last remaining admin.
</Warning>

## Performance & Scale Notes

- Scope changes are individual API calls. If you need to change multiple members' scopes, batch the operations with appropriate error handling for partial failures.
- Real-time scope change events are delivered to all group members, so large groups may generate significant event traffic.
- Consider caching member scopes locally and updating the cache when scope change events are received to reduce API calls.
- The scope change operation is atomic - it either succeeds completely or fails without partial changes.

## Security Notes

<Note>
**Auth Key** is acceptable for POC/development only. For production, use **Auth Token** created server-side via the CometChat REST API.
</Note>

<Warning>
**Server-Side Role Management**: For production applications, consider implementing server-side role management workflows. This allows you to enforce business rules, maintain audit logs, and implement approval processes before scope changes are made.
</Warning>

<Warning>
**Audit Trail**: CometChat stores scope changes as Action messages in the group history. For compliance requirements, consider maintaining your own audit log of role changes with additional context like reason for promotion/demotion.
</Warning>
