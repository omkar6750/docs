## Summary

Interactive Messages in the CometChat JavaScript SDK enable you to send rich, interactive content within chat messages, such as embedded forms, cards, and custom interactive elements. Users can interact directly with these messages without leaving the chat interface, enhancing engagement and enabling workflows like surveys, feedback collection, and action buttons.

## When to Use This

- When building in-chat forms for surveys, feedback, or data collection
- When creating interactive cards with action buttons (e.g., approve/reject workflows)
- When implementing custom interactive elements like polls or quizzes
- When tracking user interactions and goal completion within messages
- When enabling sender interaction with their own interactive messages

## Prerequisites

- **SDK**: `@cometchat/chat-sdk-javascript` v4.x
- **State**: User must be logged in via `CometChat.login()` before sending or receiving interactive messages
- **Receiver**: The recipient user (UID) or group (GUID) must exist in CometChat

## Minimal Example

<Tabs>
<Tab title="JavaScript">
```javascript
const receiverID = "UID";
const receiverType = CometChat.RECEIVER_TYPE.USER;

const interactiveData = {
  title: "Quick Survey",
  formFields: [
    {
      elementType: "textInput",
      elementId: "feedback",
      label: "Your Feedback",
      placeholder: { text: "Enter your feedback here" }
    }
  ],
  submitElement: {
    elementType: "button",
    elementId: "submitBtn",
    buttonText: "Submit"
  }
};

const interactiveMessage = new CometChat.InteractiveMessage(
  receiverID,
  receiverType,
  "form",
  interactiveData
);

CometChat.sendInteractiveMessage(interactiveMessage).then(
  (message) => {
    console.log("Interactive message sent:", message);
  },
  (error) => {
    console.log("Error sending interactive message:", error);
  }
);
```

## Step-by-Step

### Step 1: Understand InteractiveMessage Structure

An `InteractiveMessage` is a specialized chat message that encapsulates interactive content. It contains the following properties:

| Parameter              | Type    | Required | Default | Description                                                                                                                                                       |
| ---------------------- | ------- | -------- | ------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| receiverId             | string  | Yes      | -       | The UID or GUID of the recipient                                                                                                                                  |
| receiverType           | string  | Yes      | -       | The type of the receiver: `CometChat.RECEIVER_TYPE.USER` or `CometChat.RECEIVER_TYPE.GROUP` |
| messageType            | string  | Yes      | -       | The type of the interactive message (e.g., "form", "card")                                                                                                        |
| interactiveData        | object  | Yes      | -       | A JSON object holding structured data for the interactive element                                                                                                 |
| allowSenderInteraction | boolean | No       | false   | Whether the message sender can interact with the message                                                                                                          |
| interactionGoal        | object  | No       | none    | An InteractionGoal object defining the intended outcome of interacting with the message                                                                           |

### Step 5: Set Interaction Goals

Define goals to track when users complete specific interactions:

<Tabs>
<Tab title="JavaScript">
```javascript
// Goal: Any interaction completes the goal
const anyInteractionGoal = new CometChat.InteractionGoal(
  CometChat.CometChatConstants.INTERACTION_TYPE_ANY,
  []
);

// Goal: Any of specific interactions completes the goal
const anyOfGoal = new CometChat.InteractionGoal(
  CometChat.CometChatConstants.INTERACTION_TYPE_ANY_OF,
  ["submitButton", "cancelButton"]
);

// Goal: All specified interactions must occur
const allOfGoal = new CometChat.InteractionGoal(
  CometChat.CometChatConstants.INTERACTION_TYPE_ALL_OF,
  ["name", "gender", "submitButton"]
);

// Goal: Never completed (for informational messages)
const noneGoal = new CometChat.InteractionGoal(
  CometChat.CometChatConstants.INTERACTION_TYPE_NONE,
  []
);

// Apply goal to message
interactiveMessage.setInteractionGoal(anyOfGoal);
```
</Tab>

## API Details

### Method Signature

```
CometChat.sendInteractiveMessage(interactiveMessage: InteractiveMessage): Promise<InteractiveMessage>
```

### InteractiveMessage Constructor

```
new CometChat.InteractiveMessage(receiverId, receiverType, messageType, interactiveData)
```

### Parameters

| Parameter       | Type   | Required | Description                                                                 |
| --------------- | ------ | -------- | --------------------------------------------------------------------------- |
| receiverId      | string | Yes      | The UID of the user or GUID of the group receiving the message              |
| receiverType    | string | Yes      | `CometChat.RECEIVER_TYPE.USER` or `CometChat.RECEIVER_TYPE.GROUP`           |
| messageType     | string | Yes      | The type of interactive message (e.g., "form", "card", custom types)        |
| interactiveData | object | Yes      | JSON object containing the interactive element structure                    |

### InteractiveMessage Methods

| Method                        | Parameters           | Description                                           |
| ----------------------------- | -------------------- | ----------------------------------------------------- |
| setAllowSenderInteraction     | boolean              | Enable/disable sender interaction with their message  |
| setInteractionGoal            | InteractionGoal      | Set the goal for tracking interaction completion      |
| getInteractiveData            | -                    | Returns the interactive data object                   |
| getAllowSenderInteraction     | -                    | Returns whether sender interaction is allowed         |
| getInteractionGoal            | -                    | Returns the interaction goal                          |

### InteractionGoal Constructor

```
new CometChat.InteractionGoal(type, elementIds)
```

| Parameter  | Type     | Description                                                    |
| ---------- | -------- | -------------------------------------------------------------- |
| type       | string   | Goal type from CometChatConstants (ANY, ANY_OF, ALL_OF, NONE)  |
| elementIds | string[] | Array of element IDs to track for goal completion              |

### Interaction Goal Types

| Goal Type                                        | Description                                              |
| ------------------------------------------------ | -------------------------------------------------------- |
| `CometChatConstants.INTERACTION_TYPE_ANY`        | Goal completes with any interaction                      |
| `CometChatConstants.INTERACTION_TYPE_ANY_OF`     | Goal completes when any specified interaction occurs     |
| `CometChatConstants.INTERACTION_TYPE_ALL_OF`     | Goal completes when all specified interactions occur     |
| `CometChatConstants.INTERACTION_TYPE_NONE`       | Goal never completes                                     |

### Return Value

Returns a `Promise<CometChat.InteractiveMessage>` that resolves to the sent interactive message object containing:
- Message ID
- Sender information
- Receiver information
- Interactive data
- Interaction goal
- Timestamps

### Errors

| Error Code                  | Description                                    | Resolution                                      |
| --------------------------- | ---------------------------------------------- | ----------------------------------------------- |
| ERR_NOT_LOGGED_IN           | User is not logged in                          | Call `CometChat.login()` before sending         |
| ERR_INVALID_RECEIVER        | Invalid receiver ID or type                    | Verify UID/GUID exists and type is correct      |
| ERR_INVALID_MESSAGE_TYPE    | Invalid message type provided                  | Use valid message type string                   |
| ERR_EMPTY_INTERACTIVE_DATA  | Interactive data is empty or invalid           | Provide valid interactive data object           |

## Events & Real-time Behavior

### Registering Listeners

Register a message listener to receive interactive messages and track goal completions:

<Tabs>
<Tab title="JavaScript">
```javascript
const listenerId = "interactive-message-listener";

CometChat.addMessageListener(
  listenerId,
  new CometChat.MessageListener({
    onInteractiveMessageReceived: (message) => {
      console.log("Interactive message received:", message);
      // Handle the interactive message display
      const interactiveData = message.getInteractiveData();
      console.log("Interactive data:", interactiveData);
    },
    onInteractionGoalCompleted: (receipt) => {
      console.log("Interaction goal completed:", receipt);
      // Handle goal completion (e.g., update UI, trigger workflow)
      const messageId = receipt.getMessageId();
      const interactions = receipt.getInteractions();
      console.log("Completed for message:", messageId);
    }
  })
);
```
</Tab>
<Tab title="TypeScript">
```typescript
const listenerId: string = "interactive-message-listener";

CometChat.addMessageListener(
  listenerId,
  new CometChat.MessageListener({
    onInteractiveMessageReceived: (message: CometChat.InteractiveMessage) => {
      console.log("Interactive message received:", message);
      // Handle the interactive message display

      console.log("Interactive data:", interactiveData);
    },
    onInteractionGoalCompleted: (receipt: CometChat.InteractionReceipt) => {
      console.log("Interaction goal completed:", receipt);
      // Handle goal completion (e.g., update UI, trigger workflow)
      const messageId: number = receipt.getMessageId();
      const interactions: CometChat.Interaction[] = receipt.getInteractions();
      console.log("Completed for message:", messageId);
    }
  })
);
```

### Interaction Object

When users interact with an interactive message, an `Interaction` object is created:

| Property     | Type   | Description                                        |
| ------------ | ------ | -------------------------------------------------- |
| elementId    | string | Identifier of the interacted element               |
| interactedAt | number | Unix timestamp of when the interaction occurred    |

## Edge Cases & Pitfalls

<Warning>
**Pitfall 1: Missing Listener Cleanup**

Failing to remove message listeners when components unmount or pages navigate away can cause memory leaks and duplicate event handling. Always call `CometChat.removeMessageListener(listenerId)` in cleanup logic.
</Warning>

<Warning>
**Pitfall 2: Duplicate Listener IDs**

Using the same `listenerId` for multiple listeners will overwrite previous listeners. Always use unique listener IDs (e.g., include component instance ID or UUID) to avoid losing event handlers.
</Warning>

<Warning>
**Pitfall 3: Sender Interaction Disabled by Default**

By default, `allowSenderInteraction` is `false`, meaning the sender cannot interact with their own interactive message. If you need senders to test or interact with their messages, explicitly call `setAllowSenderInteraction(true)`.
</Warning>

<Warning>
**Pitfall 4: Invalid Element IDs in Interaction Goals**

When setting interaction goals with `INTERACTION_TYPE_ANY_OF` or `INTERACTION_TYPE_ALL_OF`, ensure the element IDs match exactly with those defined in your `interactiveData`. Mismatched IDs will prevent goal completion from being detected.
</Warning>

<Warning>
**Pitfall 5: Large Interactive Data Objects**

Very large `interactiveData` objects may impact message delivery performance. Keep interactive data concise and avoid embedding large base64 images or excessive form fields. Consider pagination for complex forms.
</Warning>

## Security Notes

<Note>
**Auth Key** is acceptable for POC/development only. For production, use **Auth Token** created server-side via the CometChat REST API.
</Note>

- Validate interactive data on your server before processing user submissions
- Sanitize any user input from interactive forms to prevent XSS attacks
- Use HTTPS URLs in action configurations to ensure secure navigation
- Consider rate limiting interactive message submissions to prevent abuse
