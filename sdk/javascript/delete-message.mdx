## Summary

The CometChat JavaScript SDK allows you to delete previously sent messages. This page covers how to delete messages, receive real-time delete events, and handle missed delete events when your app was offline.

## When to Use This

- When allowing users to remove messages they no longer want visible
- When implementing message deletion functionality in your chat UI
- When building moderation features that allow admins to remove inappropriate content
- When users need to retract accidentally sent messages
- When syncing deleted messages across multiple devices or sessions

## Prerequisites

- **SDK**: `@cometchat/chat-sdk-javascript` v4.x
- **State**: User must be logged in via `CometChat.login()` before deleting messages
- **Permissions**: User must be the message sender, or a group admin/moderator for group messages
- **Message**: The message to delete must exist and have a valid message ID

## Minimal Example

<Tabs>
<Tab title="JavaScript">
```javascript
const messageId = MESSAGE_ID;

CometChat.deleteMessage(messageId).then(
  (message) => {
    console.log("Message deleted successfully:", message);
  },
  (error) => {
    console.log("Message deletion failed:", error);
  }
);
```
</Tab>

```typescript
  }

```

## Step-by-Step

### Delete a Message

*In other words, as a sender, how do I delete a message?*

To delete a message, use the `deleteMessage()` method. This method takes the message ID of the message to be deleted.

<Tabs>
<Tab title="JavaScript">
```javascript
const messageId = MESSAGE_ID;

CometChat.deleteMessage(messageId).then(
  (message) => {
    console.log("Message deleted:", message);
    console.log("Deleted at:", message.getDeletedAt());
    console.log("Deleted by:", message.getDeletedBy());
  },
  (error) => {
    console.log("Message delete failed with error:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript
const messageId: number = MESSAGE_ID;

CometChat.deleteMessage(messageId).then(
  (message: CometChat.BaseMessage) => {
    console.log("Message deleted:", message);
    console.log("Deleted at:", message.getDeletedAt());
    console.log("Deleted by:", message.getDeletedBy());
  },
  (error: CometChat.CometChatException) => {
    console.log("Message delete failed with error:", error);
  }
);
```
</Tab>
</Tabs>

The deleted message object returned will contain:
- `deletedAt` - Timestamp of when the message was deleted
- `deletedBy` - UID of the user who deleted the message

These two fields can be used to identify if a message is deleted while iterating through a list of messages.

### Delete Permissions

By default, CometChat allows certain roles to delete messages:

| User Role       | Conversation Type       | Deletion Capabilities          |
| --------------- | ----------------------- | ------------------------------ |
| Message Sender  | One-on-One Conversation | Messages they have sent        |
| Message Sender  | Group Conversation      | Messages they have sent        |
| Group Admin     | Group Conversation      | All messages in the group      |
| Group Moderator | Group Conversation      | All messages in the group      |

### Real-time Message Delete Events

*In other words, as a recipient, how do I know when someone deletes a message when my app is running?*

To receive real-time events for message deletions, override the `onMessageDeleted()` method of the `MessageListener` class.

#### Registering the Listener

<Tabs>
<Tab title="JavaScript">
```javascript
const listenerID = "UNIQUE_LISTENER_ID";

CometChat.addMessageListener(
  listenerID,
  new CometChat.MessageListener({
    onMessageDeleted: (message) => {
      console.log("Message deleted:", message);
      console.log("Deleted at:", message.getDeletedAt());
      console.log("Deleted by:", message.getDeletedBy());
    }
  })
);
```
</Tab>
<Tab title="TypeScript">
```typescript
const listenerID: string = "UNIQUE_LISTENER_ID";

CometChat.addMessageListener(
  listenerID,
  new CometChat.MessageListener({
    onMessageDeleted: (message: CometChat.BaseMessage) => {
      console.log("Message deleted:", message);
      console.log("Deleted at:", message.getDeletedAt());
      console.log("Deleted by:", message.getDeletedBy());
    }
  })
);
```
</Tab>
</Tabs>

#### Removing the Listener

Always remove listeners when they are no longer needed to prevent memory leaks:

<Tabs>
<Tab title="JavaScript">
```javascript
const listenerID = "UNIQUE_LISTENER_ID";

CometChat.removeMessageListener(listenerID);
```
</Tab>
<Tab title="TypeScript">
```typescript
const listenerID: string = "UNIQUE_LISTENER_ID";

CometChat.removeMessageListener(listenerID);
```
</Tab>
</Tabs>

### Missed Message Delete Events

*In other words, as a recipient, how do I know if someone deleted a message when my app was not running?*

When you retrieve the list of previous messages, any deleted messages will have the `deletedAt` and `deletedBy` fields set. Additionally, an `Action` message is created to inform you about the deletion.

For example, if a conversation has 100 messages and message ID 50 was deleted while your app was offline:
- Message 50 will have `deletedAt` and `deletedBy` fields populated
- Message 101 will be an `Action` message indicating the deletion

#### Action Object Fields

For message delete events, the `Action` object contains:

| Field       | Value                                                    |
| ----------- | -------------------------------------------------------- |
| `action`    | `deleted`                                                |
| `actionOn`  | Updated message object which was deleted                 |
| `actionBy`  | User object of the user who deleted the message          |
| `actionFor` | User/Group object of the conversation receiver           |

## API Details

### Method Signature

```
CometChat.deleteMessage(messageId: number): Promise<BaseMessage>
```

### Parameters

| Parameter | Type   | Required | Description                           |
| --------- | ------ | -------- | ------------------------------------- |
| messageId | number | Yes      | The ID of the message to delete       |

### Return Value

Returns a `Promise<BaseMessage>` that resolves to the deleted message object containing:
- All original message properties
- `deletedAt` - Timestamp of the deletion
- `deletedBy` - UID of the user who performed the deletion

### Errors

| Error Code                  | Description                                    | Resolution                                      |
| --------------------------- | ---------------------------------------------- | ----------------------------------------------- |
| ERR_MESSAGE_NOT_FOUND       | Message with specified ID does not exist       | Verify the message ID is correct                |
| ERR_PERMISSION_DENIED       | User lacks permission to delete this message   | Ensure user is sender or group admin/moderator  |
| ERR_NOT_LOGGED_IN           | User is not logged in                          | Call CometChat.login() before deleting messages |

## Edge Cases & Pitfalls

<Warning>
**Message ID Must Be Valid**: The `messageId` parameter must be a valid numeric ID of an existing message. Passing an invalid or non-existent message ID will result in an error. Always retrieve the message ID from the message object you want to delete.
</Warning>

<Warning>
**Listener ID Uniqueness**: Each listener must have a unique `listenerID`. If you register multiple listeners with the same ID, only the last one will be active. Use descriptive, unique IDs like `"chat-screen-delete-listener"` to avoid conflicts.
</Warning>

<Warning>
**Memory Leaks from Listeners**: Always remove message listeners when the component unmounts or when navigating away from the chat screen. Failing to remove listeners can cause memory leaks and duplicate event handling.
</Warning>

<Warning>
**Deleted Messages in History**: Deleted messages are not removed from message history. Instead, they remain with `deletedAt` and `deletedBy` fields populated. Your UI should check these fields and display an appropriate "message deleted" placeholder instead of the original content.
</Warning>

<Note>
In order to delete a message, you need to be either the sender of the message or the admin/moderator of the group in which the message was sent.
</Note>
