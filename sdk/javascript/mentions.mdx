## Summary

The CometChat JavaScript SDK enables you to mention specific users within messages using the `<@uid:UID>` format. Mentions allow users to directly reference and notify other participants in a conversation, making it easier to engage specific individuals in group discussions or direct messages.

## When to Use This

- When building @mention functionality to tag specific users in messages
- When implementing notification systems that alert users when they are mentioned
- When creating group chat features where users need to direct messages to specific participants
- When building search or filter functionality to find messages where specific users are mentioned
- When displaying mentioned user information like tags or blocked status

## Prerequisites

1. **SDK**: `@cometchat/chat-sdk-javascript` v4.x installed and configured
2. **State**: User must be logged in via `CometChat.login()` before sending or fetching mentioned messages
3. **Receiver**: The recipient user (UID) or group (GUID) must exist in CometChat
4. **Mentioned User**: The user being mentioned must exist in CometChat with a valid UID

## Minimal Example

<Tabs>
<Tab title="JavaScript">
```javascript
const receiverID = "UID";
const messageText = "Hello, <@uid:cometchat-uid-1>";
const receiverType = CometChat.RECEIVER_TYPE.USER;

const textMessage = new CometChat.TextMessage(
  receiverID,
  messageText,
  receiverType
);

CometChat.sendMessage(textMessage).then(
  (message) => {
    console.log("Message sent successfully:", message);
  },
  (error) => {
    console.log("Message sending failed:", error);
  }
);
```

### Step 2: Fetch Messages with Mention Tag Info

To retrieve messages where users are mentioned along with the tags of the mentioned users, use the `mentionsWithTagInfo(true)` method on the `MessagesRequestBuilder`.

<Note>
Advanced mention filtering options like `mentionsWithTagInfo` and `mentionsWithBlockedInfo` are available on specific plans. Check your plan details in the [CometChat Dashboard](https://app.cometchat.com).
</Note>

<Tabs>
<Tab title="JavaScript">
```javascript
let UID = "UID";
let limit = 30;

var messagesRequest = new CometChat.MessagesRequestBuilder()
  .setUID(UID)
  .setLimit(limit)
  .mentionsWithTagInfo(true)
  .build();

messagesRequest.fetchPrevious().then(
  (messages) => {
    messages.forEach((eachMessage) => {
      eachMessage.getMentionedUsers().forEach((eachMentionedUser) => {
        console.log(eachMentionedUser.getTags());
      });
    });
  },
  (error) => {
    console.log("Message fetching failed with error:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript
let UID: string = "UID";
let limit: number = 30;
let messagesRequest: CometChat.MessagesRequest =
  new CometChat.MessagesRequestBuilder()
    .setUID(UID)
    .setLimit(limit)
    .mentionsWithTagInfo(true)
    .build();

messagesRequest.fetchPrevious().then(
  (messages: CometChat.BaseMessage[]) => {
    messages.forEach((eachMessage: CometChat.BaseMessage) => {
      eachMessage
        .getMentionedUsers()
        .forEach((eachMentionedUser: CometChat.User) => {
          console.log(eachMentionedUser.getTags());
        });
    });
  },
  (error: CometChat.CometChatException) => {
    console.log("Message fetching failed with error:", error);
  }
);
```
</Tab>
</Tabs>

```javascript
        console.log("hasBlockedMe: " + eachMentionedUser.getHasBlockedMe());
```

### Step 4: Get Mentioned Users from a Message

To retrieve the list of users mentioned in a particular message, use the `getMentionedUsers()` method on the message object.

<Tabs>
<Tab title="JavaScript">
```javascript
// After receiving or fetching a message
const mentionedUsers = message.getMentionedUsers();
console.log("Mentioned users:", mentionedUsers);
```
</Tab>
<Tab title="TypeScript">
```typescript
// After receiving or fetching a message
const mentionedUsers: CometChat.User[] = message.getMentionedUsers();
```

## API Details

### Mention Format

Use the `<@uid:UID>` format to mention users in message text.

| Format | Description | Example |
|--------|-------------|---------|
| `<@uid:UID>` | Mention a specific user by their UID | `<@uid:cometchat-uid-1>` |

### MessagesRequestBuilder Methods

| Method | Type | Default | Description |
|--------|------|---------|-------------|
| `mentionsWithTagInfo(value)` | `boolean` | `false` | When `true`, fetches messages with mentioned users and their tags |
| `mentionsWithBlockedInfo(value)` | `boolean` | `false` | When `true`, fetches messages with mentioned users and their blocked relationship status |

### Message Methods

| Method | Return Type | Description |
|--------|-------------|-------------|
| `getMentionedUsers()` | `User[]` | Returns an array of users mentioned in the message, or empty array if none |

### User Methods (for mentioned users)

| Method | Return Type | Description |
|--------|-------------|-------------|
| `getTags()` | `string[]` | Returns the tags associated with the user (requires `mentionsWithTagInfo(true)`) |
| `getBlockedByMe()` | `boolean` | Returns `true` if the logged-in user has blocked this user (requires `mentionsWithBlockedInfo(true)`) |
| `getHasBlockedMe()` | `boolean` | Returns `true` if this user has blocked the logged-in user (requires `mentionsWithBlockedInfo(true)`) |

## Edge Cases & Pitfalls

<Warning>
**Invalid UID Format**: Ensure the UID in the mention format `<@uid:UID>` matches an existing user. Mentioning a non-existent UID will not throw an error but the mention will not resolve to a user.
</Warning>

<Warning>
**Missing Tag/Blocked Info**: By default, `getMentionedUsers()` returns basic user information. To get tags or blocked status, you must explicitly set `mentionsWithTagInfo(true)` or `mentionsWithBlockedInfo(true)` when building the messages request.
</Warning>

<Warning>
**Empty Mentioned Users Array**: If no users are mentioned in a message, `getMentionedUsers()` returns an empty array, not `null`. Always check the array length before iterating.
</Warning>

<Warning>
**Plan Restrictions**: Advanced mention filtering options (`mentionsWithTagInfo`, `mentionsWithBlockedInfo`) may require specific CometChat plans. Check your plan details in the dashboard if these features don't work as expected.
</Warning>

## Performance & Scale Notes

- **Pagination**: Use `setLimit()` to control the number of messages fetched per request. The default and maximum limit is typically 100 messages.
- **Fetch Direction**: Use `fetchPrevious()` to load older messages (for scrolling up) or `fetchNext()` for newer messages.
- **Batching**: When processing many messages with mentions, consider batching your UI updates to avoid performance issues.
- **Caching**: Cache mentioned user information locally to reduce redundant API calls when displaying the same users across multiple messages.

## Security Notes

<Note>
**Auth Key** is acceptable for POC/development only. For production applications, use **Auth Token** created server-side via the CometChat REST API.
</Note>

- Mentioned users can only be resolved if they exist in your CometChat app
- Blocked relationship information respects user privacy settings
- User tags are only visible if the requesting user has appropriate permissions
