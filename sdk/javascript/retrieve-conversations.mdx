title: "Retrieve Conversations"
---

## Summary

The CometChat JavaScript SDK enables you to retrieve conversations for the logged-in user, providing the last message for every one-on-one and group conversation. This makes it easy to build a "Recent Chats" list, fetch specific conversations, tag conversations for organization, and convert real-time messages into conversation objects.

## When to Use This

- When building a recent chats or inbox view showing all user conversations
- When implementing conversation search functionality to find specific chats
- When organizing conversations with tags (e.g., archived, starred, important)
- When fetching a specific conversation's details by user or group ID
- When converting incoming real-time messages to conversation objects for UI updates

## Prerequisites

1. **SDK**: `@cometchat/chat-sdk-javascript` v4.x installed and initialized
2. **Authentication**: User must be logged in via `CometChat.login()`
3. **Plan**: Search keyword and unread filters require **Advanced** or **Custom** plan with **Conversation & Advanced Search** enabled
4. **Dashboard**: Enable Conversation & Advanced Search in [CometChat Dashboard](https://app.cometchat.com) → Chats → Settings → General Configuration

## Minimal Example

<Tabs>
<Tab title="JavaScript">
```javascript
const limit = 30;
const conversationsRequest = new CometChat.ConversationsRequestBuilder()
  .setLimit(limit)
  .build();

conversationsRequest.fetchNext().then(
  (conversationList) => {
    console.log("Conversations list received:", conversationList);
  },
  (error) => {
    console.log("Conversations list fetching failed with error:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript
const limit: number = 30;
const conversationsRequest: CometChat.ConversationsRequest = new CometChat.ConversationsRequestBuilder()
  .setLimit(limit)
  .build();

conversationsRequest.fetchNext().then(
  (conversationList: CometChat.Conversation[]) => {
    console.log("Conversations list received:", conversationList);
  },
  (error: CometChat.CometChatException) => {
    console.log("Conversations list fetching failed with error:", error);
  }
);
```
</Tab>
</Tabs>

## Step-by-Step

### Retrieve List of Conversations

To fetch the list of conversations, use the `ConversationsRequest` class. Create an object using the `ConversationsRequestBuilder` class, which allows you to set various parameters for filtering conversations.

#### Set Limit

This method sets the number of conversations that should be fetched in a single iteration.

<Tabs>
<Tab title="JavaScript">
```javascript
let limit = 30;
let conversationRequest = new CometChat.ConversationsRequestBuilder()
  .setLimit(limit)
  .build();
```
</Tab>
<Tab title="TypeScript">
```typescript
let limit: number = 30;
let conversationRequest: CometChat.ConversationsRequest = new CometChat.ConversationsRequestBuilder()
  .setLimit(limit)
  .build();
```
</Tab>
</Tabs>

#### Set Conversation Type

This method can be used to fetch user or group conversations specifically. The `conversationType` variable can hold one of the below two values:

- `user` - Only fetches user conversation.
- `group` - Only fetches group conversations.

If none is set, the list of conversations will include both user and group conversations.

<Tabs>
<Tab title="JavaScript">
```javascript
let limit = 30;
let conversationType = "group";
let conversationRequest = new CometChat.ConversationsRequestBuilder()
  .setLimit(limit)
  .setConversationType(conversationType)
  .build();
```
</Tab>
<Tab title="TypeScript">
```typescript
let limit: number = 30;
let conversationType: string = "group";
let conversationRequest: CometChat.ConversationsRequest = new CometChat.ConversationsRequestBuilder()
  .setLimit(limit)
  .setConversationType(conversationType)
  .build();
```
</Tab>
</Tabs>

#### Fetch Conversations

Once all the parameters are set to the builder class, call the `build()` method to get the object of the `ConversationsRequest` class. Then call the `fetchNext()` method to retrieve the conversations.

A maximum of only 50 conversations can be fetched at once.

<Tabs>
<Tab title="JavaScript">
```javascript
let limit = 30;
let conversationsRequest = new CometChat.ConversationsRequestBuilder()
  .setLimit(limit)
  .build();

conversationsRequest.fetchNext().then(
  (conversationList) => {
    console.log("Conversations list received:", conversationList);
  },
  (error) => {
    console.log("Conversations list fetching failed with error:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript
let limit: number = 30;
let conversationsRequest: CometChat.ConversationsRequest = new CometChat.ConversationsRequestBuilder()
  .setLimit(limit)
  .build();

  (conversationList: CometChat.Conversation[]) => {
    console.log("Conversations list received:", conversationList);

  }
);
```
</Tab>

### Tag Conversation

To tag a specific conversation, use the `tagConversation()` method. This method accepts three parameters:

1. `conversationWith`: UID/GUID of the user/group whose conversation you want to tag.
2. `conversationType`: Either `user` or `group`.
3. `tags`: A list of tags you want to add to the conversation.

<Tabs>
<Tab title="JavaScript">
```javascript
let tags = ["archivedChat"];

CometChat.tagConversation("UID", "user", tags).then(
  (conversation) => {
    console.log("Conversation tagged:", conversation);
  },
  (error) => {
    console.log("Error while tagging conversation:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript
let conversationWith: string = "UID";
let tags: Array<String> = ["archivedChat"];
let conversationType: string = "user";

CometChat.tagConversation(conversationWith, conversationType, tags).then(
  (conversation: CometChat.Conversation) => {
    console.log("Conversation tagged:", conversation);
  },
  (error: CometChat.CometChatException) => {
    console.log("Error while tagging conversation:", error);
  }
);
```
</Tab>
</Tabs>

<Note>
The tags for conversations are one-way. This means that if user A tags a conversation with user B, that tag will be applied to that conversation only for user A.
</Note>

### Retrieve Single Conversation

To fetch a specific conversation, use the `getConversation` method. This method accepts two parameters:

1. `conversationWith`: UID/GUID of the user/group whose conversation you want to fetch.
2. `conversationType`: Either `user` or `group`.

<Tabs>
<Tab title="JavaScript">
```javascript

  (conversation) => {
    console.log("Conversation retrieved:", conversation);
  },
  (error) => {
    console.log("Error while fetching conversation:", error);
  }
);
```
</Tab>

```typescript
let conversationType: string = "user";

```
</Tab>

### Convert Messages to Conversations

For real-time messages, you will always receive `Message` objects and not `Conversation` objects. Use the `getConversationFromMessage()` method of the `CometChatHelper` class to convert a message to a conversation object.

<Tabs>
<Tab title="JavaScript">
```javascript
CometChat.CometChatHelper.getConversationFromMessage(message).then(
  (conversation) => {
    console.log("Conversation Object:", conversation);
  },
  (error) => {
    console.log("Error while converting message object:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript
let message: CometChat.TextMessage | CometChat.MediaMessage | CometChat.CustomMessage;

CometChat.CometChatHelper.getConversationFromMessage(message).then(
  (conversation: CometChat.Conversation) => {
    console.log("Conversation Object:", conversation);
  },
  (error: CometChat.CometChatException) => {
    console.log("Error while converting message object:", error);
  }
);
```
</Tab>
</Tabs>

<Note>
While converting the `Message` object to the `Conversation` object, the `unreadMessageCount` and `tags` will not be available in the `Conversation` object. The unread message count needs to be managed in your client-side code.
</Note>

## API Details

### ConversationsRequestBuilder Methods

| Method | Parameters | Description |
|--------|------------|-------------|
| `setLimit(limit)` | `limit: number` | Sets the number of conversations to fetch per request (max 50) |
| `setConversationType(type)` | `type: string` | Filter by `user` or `group` conversations |
| `withUserAndGroupTags(include)` | `include: boolean` | Include user/group tags in response |
| `setUserTags(tags)` | `tags: string[]` | Filter user conversations by tags |
| `setGroupTags(tags)` | `tags: string[]` | Filter group conversations by tags |
| `withTags(include)` | `include: boolean` | Include conversation tags in response |
| `setTags(tags)` | `tags: string[]` | Filter conversations by tags |
| `setIncludeBlockedUsers(include)` | `include: boolean` | Include blocked user conversations |
| `setWithBlockedInfo(include)` | `include: boolean` | Include blocked info in response |
| `setSearchKeyword(keyword)` | `keyword: string` | Search by user/group name (plan-gated) |
| `setUnread(unread)` | `unread: boolean` | Filter to unread conversations only (plan-gated) |
| `setHideAgentic(hide)` | `hide: boolean` | Exclude AI agent conversations |
| `setOnlyAgentic(only)` | `only: boolean` | Include only AI agent conversations |
| `build()` | - | Returns `ConversationsRequest` object |

### Conversation Object Fields

| Field | Type | Description |
|-------|------|-------------|
| `conversationId` | `string` | Unique ID of the conversation |
| `conversationType` | `string` | Type of conversation (`user` or `group`) |
| `lastMessage` | `BaseMessage` | Last message in the conversation |
| `conversationWith` | `User \| Group` | User or Group object with details |
| `unreadMessageCount` | `number` | Count of unread messages |
| `tags` | `string[]` | Tags associated with the conversation |

### Return Values

- `fetchNext()`: Returns `Promise<Conversation[]>` - Array of conversation objects
- `getConversation()`: Returns `Promise<Conversation>` - Single conversation object
- `tagConversation()`: Returns `Promise<Conversation>` - Updated conversation object
- `getConversationFromMessage()`: Returns `Promise<Conversation>` - Converted conversation object

## Edge Cases & Pitfalls

<Warning>
**Pagination Limit**: A maximum of 50 conversations can be fetched per request. Always implement pagination using `fetchNext()` for complete conversation lists.
</Warning>

<Warning>
**Unread Count Not Available in Conversion**: When converting a `Message` to a `Conversation` using `getConversationFromMessage()`, the `unreadMessageCount` and `tags` fields will not be populated. You must manage unread counts client-side.
</Warning>

<Warning>
**Mutually Exclusive Agentic Filters**: The `setHideAgentic()` and `setOnlyAgentic()` methods cannot be used together. Using both in the same request builder will cause unexpected behavior.
</Warning>

<Warning>
**One-Way Conversation Tags**: Tags applied to a conversation are user-specific. If User A tags a conversation with User B, User B will not see that tag on their side.
</Warning>

<Warning>
**Plan-Gated Features**: The `setSearchKeyword()` and `setUnread()` filters require the **Conversation & Advanced Search** add-on. Ensure your plan supports these features before implementation.
</Warning>

## Performance & Scale Notes

- **Pagination**: Use `setLimit()` with values between 1-50 and call `fetchNext()` repeatedly to paginate through all conversations
- **Caching**: Consider caching conversation lists locally and updating incrementally using real-time listeners
- **Filtering**: Use specific filters (conversation type, tags) to reduce response payload size
- **Search**: The search keyword feature performs server-side filtering, which is more efficient than client-side filtering for large conversation lists

## Security Notes

<Note>
**Auth Key** is acceptable for POC/development only. For production applications, use **Auth Token** created server-side via the CometChat REST API to authenticate users securely.
</Note>
