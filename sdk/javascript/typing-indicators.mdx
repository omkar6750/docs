## Summary

Typing Indicators in the CometChat JavaScript SDK enable real-time display of when users are actively composing messages. This feature enhances chat experiences by showing "User is typing..." notifications, helping users know when to expect a response and creating more engaging conversations.

## When to Use This

- When you want to show "User is typing..." indicators in one-on-one chats
- When displaying typing notifications in group conversations
- When building real-time chat interfaces that need activity feedback
- When you want to improve user engagement by showing conversation activity
- When implementing chat UX patterns similar to popular messaging apps

## Prerequisites

- **SDK**: `@cometchat/chat-sdk-javascript` v4.x
- **State**: User must be logged in via `CometChat.login()`
- **Dashboard**: Ensure your CometChat app is created in the [CometChat Dashboard](https://app.cometchat.com)

## Minimal Example

<Tabs>
<Tab title="JavaScript">
```javascript
// Send typing indicator to a user
let receiverId = "UID";
let receiverType = CometChat.RECEIVER_TYPE.USER;

let typingNotification = new CometChat.TypingIndicator(receiverId, receiverType);
CometChat.startTyping(typingNotification);

// Listen for typing indicators
let listenerId = "UNIQUE_LISTENER_ID";
CometChat.addMessageListener(
  listenerId,
  new CometChat.MessageListener({
    onTypingStarted: (typingIndicator) => {
      console.log("User is typing:", typingIndicator);
    },
    onTypingEnded: (typingIndicator) => {
      console.log("User stopped typing:", typingIndicator);
    }
  })
);
```
</Tab>

### Step 3: Add Custom Metadata (Optional)

You can attach custom data to typing indicators using the `metadata` field. This is useful for sending additional context like which message is being replied to or the type of content being composed.

<Tabs>
<Tab title="JavaScript">
```javascript
let receiverId = "UID";
let receiverType = CometChat.RECEIVER_TYPE.USER;

let typingNotification = new CometChat.TypingIndicator(receiverId, receiverType);

// Add custom metadata
let metadata = {
  replyingTo: "MESSAGE_ID",
  contentType: "image"
};
typingNotification.setMetadata(metadata);

CometChat.startTyping(typingNotification);
```
</Tab>
<Tab title="TypeScript">

### Step 4: Register Typing Indicator Listener

To receive typing indicators from other users, register a `MessageListener` with `onTypingStarted` and `onTypingEnded` callbacks.

<Tabs>
<Tab title="JavaScript">
```javascript
let listenerId = "UNIQUE_LISTENER_ID";

CometChat.addMessageListener(
  listenerId,
  new CometChat.MessageListener({
    onTypingStarted: (typingIndicator) => {
      let sender = typingIndicator.getSender();
      let receiverType = typingIndicator.getReceiverType();
      
      if (receiverType === CometChat.RECEIVER_TYPE.USER) {
        console.log(`${sender.getName()} is typing...`);
      } else {
        console.log(`${sender.getName()} is typing in group...`);
      }
    },
    onTypingEnded: (typingIndicator) => {
      let sender = typingIndicator.getSender();
      console.log(`${sender.getName()} stopped typing`);
    }
  })
);
```
</Tab>
<Tab title="TypeScript">
```typescript
let listenerId: string = "UNIQUE_LISTENER_ID";

  listenerId,
  new CometChat.MessageListener({
```

## API Details

### Method Signatures

```javascript
// TypingIndicator constructor
new CometChat.TypingIndicator(receiverId: string, receiverType: string)

// Send typing indicators
CometChat.startTyping(typingIndicator: TypingIndicator): void
CometChat.endTyping(typingIndicator: TypingIndicator): void

// TypingIndicator methods
TypingIndicator.setMetadata(metadata: object): void
TypingIndicator.getMetadata(): object
TypingIndicator.getSender(): User
TypingIndicator.getReceiverId(): string
TypingIndicator.getReceiverType(): string

// Listener methods
CometChat.addMessageListener(listenerId: string, listener: MessageListener): void
CometChat.removeMessageListener(listenerId: string): void
```

### Parameters

#### TypingIndicator Constructor

| Parameter    | Type   | Required | Description                                                                 |
| ------------ | ------ | -------- | --------------------------------------------------------------------------- |
| receiverId   | string | Yes      | UID of the user or GUID of the group to send the typing indicator to        |
| receiverType | string | Yes      | `CometChat.RECEIVER_TYPE.USER` or `CometChat.RECEIVER_TYPE.GROUP`           |

#### addMessageListener

| Parameter  | Type            | Required | Description                                      |
| ---------- | --------------- | -------- | ------------------------------------------------ |
| listenerId | string          | Yes      | Unique identifier for the listener               |
| listener   | MessageListener | Yes      | MessageListener instance with callback methods   |

### Return Values

The `startTyping()` and `endTyping()` methods do not return values. They send typing indicators asynchronously.

### TypingIndicator Object Properties

| Property       | Type   | Description                                                                                                    |
| -------------- | ------ | -------------------------------------------------------------------------------------------------------------- |
| sender         | User   | User object of the person who is typing                                                                        |
| receiverId     | string | UID of the user or GUID of the group the typing indicator was sent to                                          |
| receiverType   | string | `CometChat.RECEIVER_TYPE.USER` or `CometChat.RECEIVER_TYPE.GROUP`                                              |
| metadata       | object | Custom JSON data attached to the typing indicator                                                              |

### Errors

| Error Code        | Description                       | Resolution                                     |
| ----------------- | --------------------------------- | ---------------------------------------------- |
| ERR_NOT_LOGGED_IN | User is not logged in             | Ensure user is logged in via CometChat.login() |
| ERR_INVALID_UID   | Invalid user ID provided          | Verify the UID exists in your CometChat app    |
| ERR_INVALID_GUID  | Invalid group ID provided         | Verify the GUID exists in your CometChat app   |

### Event Ordering

- `onTypingStarted` is triggered when a user begins typing in a conversation
- `onTypingEnded` is triggered when a user stops typing or sends a message
- Typing indicators are transient and not persisted—they are only delivered to currently connected users
- If the sender disconnects without calling `endTyping()`, the receiver may not receive an `onTypingEnded` event
- In group chats, all online members receive typing indicators from any member who is typing


## Edge Cases & Pitfalls

<Warning>
**Missing endTyping() Calls**: If you call `startTyping()` but forget to call `endTyping()`, the recipient may see a perpetual "typing..." indicator. Always call `endTyping()` when the user sends a message, clears the input, or navigates away from the chat.
</Warning>

<Warning>
**Duplicate Listener IDs**: Using the same `listenerId` for multiple listeners will overwrite the previous listener. Always use unique listener IDs, especially when registering listeners in multiple components. Consider using component-specific prefixes or timestamps like `"chat_screen_typing_" + Date.now()`.
</Warning>

<Warning>
**Memory Leaks from Unremoved Listeners**: Failing to remove listeners when components unmount can cause memory leaks and duplicate event handling. Always call `removeMessageListener()` in cleanup code (e.g., React's `useEffect` cleanup, Vue's `onUnmounted`, or Angular's `ngOnDestroy`).
</Warning>

<Warning>
**Throttling startTyping() Calls**: Calling `startTyping()` on every keystroke can generate excessive network traffic. Implement debouncing or throttling—for example, only send a typing indicator once every 2-3 seconds while the user is actively typing.
</Warning>

<Warning>
**No Automatic Timeout**: CometChat does not automatically send `endTyping()` after a period of inactivity. Your application should implement a timeout (e.g., 5 seconds of no keystrokes) to automatically call `endTyping()` and clear the typing indicator.
</Warning>

## Performance & Scale Notes

- Typing indicators are delivered via WebSocket for real-time performance
- Typing indicators are transient and not stored—they are only delivered to currently connected users
- In large group chats, typing indicators from multiple users may arrive frequently; consider UI strategies like "3 people are typing..." instead of listing all names
- Implement client-side debouncing to avoid sending excessive `startTyping()` calls on every keystroke
- Typing indicators do not count against message rate limits

## Security Notes

<Note>
Typing indicators are visible to all participants in a conversation. In one-on-one chats, only the recipient sees the indicator. In group chats, all group members see typing indicators from any member.
</Note>

<Note>
Custom metadata attached to typing indicators is visible to recipients. Do not include sensitive information in the metadata field.
</Note>
