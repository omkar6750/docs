## Summary

The CometChat JavaScript SDK enables users to join groups and participate in group conversations. Users can join public groups directly, password-protected groups with the correct password, and private groups only through invitations. This is essential for building community features, team collaboration, and social applications where users need to opt-in to group conversations.

## When to Use This

- When allowing users to discover and join public groups
- When implementing group invitation acceptance flows
- When building community or social features with joinable groups
- When creating team collaboration spaces users can opt into
- When implementing password-protected group access
- When building group discovery and browsing features

## Prerequisites

- **SDK**: `@cometchat/chat-sdk-javascript` v4.x
- **State**: User must be logged in via `CometChat.login()`
- **Dashboard**: Ensure your CometChat app is created in the [CometChat Dashboard](https://app.cometchat.com)
- **Group**: The group must exist and be accessible to the user
- **Group Type**: User must have appropriate access (public groups are open, password groups require password, private groups require invitation)

## Minimal Example

<Tabs>
<Tab title="JavaScript">
```javascript
let GUID = "GUID";
let groupType = CometChat.GROUP_TYPE.PUBLIC;
let password = "";
```

## Step-by-Step

### Step 1: Understand Group Types

CometChat supports three types of groups, each with different join requirements:

| Group Type | Constant                          | Join Requirement                           |
| ---------- | --------------------------------- | ------------------------------------------ |
| Public     | `CometChat.GROUP_TYPE.PUBLIC`     | Anyone can join directly                   |
| Password   | `CometChat.GROUP_TYPE.PASSWORD`   | Requires correct password to join          |
| Private    | `CometChat.GROUP_TYPE.PRIVATE`    | Can only join via invitation from a member |

### Step 2: Join a Public Group

Public groups allow any user to join without restrictions.

<Tabs>

### Step 4: Check if User Has Already Joined

CometChat tracks group membership automatically. You can check if a user has already joined a group using the `hasJoined` property on the `Group` object.

<Tabs>
<Tab title="JavaScript">
```javascript
let GUID = "GUID";

CometChat.getGroup(GUID).then(
  (group) => {
    if (group.getHasJoined()) {
      console.log("User is already a member of this group");
    } else {
      console.log("User has not joined this group yet");
    }
  },
  (error) => {
    console.log("Failed to get group:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">

## Events & Real-time Behavior

### Registering GroupListener

To receive real-time events when a user joins a group, register a `GroupListener` with the `onGroupMemberJoined` callback.

<Tabs>
<Tab title="JavaScript">
```javascript
let listenerId = "unique-join-listener";

CometChat.addGroupListener(
  listenerId,
  new CometChat.GroupListener({
    onGroupMemberJoined: (message, joinedUser, joinedGroup) => {
      console.log("User joined:", {
        message,
        joinedUser,
        joinedGroup,
      });
    },
  })
);
```
</Tab>
<Tab title="TypeScript">
```typescript
let listenerId: string = "unique-join-listener";

CometChat.addGroupListener(
  listenerId,
  new CometChat.GroupListener({
    onGroupMemberJoined: (
      message: CometChat.Action,
      joinedUser: CometChat.User,
      joinedGroup: CometChat.Group
    ) => {
      console.log("User joined:", {
        message,
        joinedUser,
        joinedGroup,
      });
    },
  })
);
```
</Tab>
</Tabs>

### Join Events in Message History

When retrieving previous messages, if a member has joined a group, the message list will contain an `Action` message. The `Action` object provides the following information:

| Property  | Description                                           |
| --------- | ----------------------------------------------------- |
| action    | The action type: `"joined"`                           |
| actionBy  | User object of the user who joined the group          |
| actionFor | Group object of the group the user joined             |

## API Details

### joinGroup()

```javascript
CometChat.joinGroup(GUID: string, groupType: string, password?: string): Promise<Group>
```

| Parameter | Type   | Required | Description                                                                                                                                                      |
| --------- | ------ | -------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| GUID      | string | Yes      | The unique identifier of the group to join                                                                                                                       |
| groupType | string | Yes      | Type of the group: `CometChat.GROUP_TYPE.PUBLIC`, `CometChat.GROUP_TYPE.PASSWORD`, or `CometChat.GROUP_TYPE.PRIVATE`                                             |
| password  | string | No       | Password for password-protected groups. Required when `groupType` is `CometChat.GROUP_TYPE.PASSWORD`                                                             |

### Return Value

Returns a `Promise<CometChat.Group>` that resolves to the `Group` object when the user successfully joins the group.

### Group Object Properties

| Property    | Type    | Description                                      |
| ----------- | ------- | ------------------------------------------------ |
| guid        | string  | Unique identifier of the group                   |
| name        | string  | Display name of the group                        |
| type        | string  | Group type (public, password, private)           |
| hasJoined   | boolean | Whether the current user has joined the group    |
| membersCount| number  | Total number of members in the group             |
| owner       | string  | UID of the group owner                           |

### Errors

| Error Code                | Description                                    | Resolution                                        |
| ------------------------- | ---------------------------------------------- | ------------------------------------------------- |
| ERR_USER_NOT_LOGGED_IN    | No user is currently logged in                 | Ensure user is logged in before joining           |
| ERR_INVALID_GUID          | The provided GUID is invalid                   | Verify the GUID format and existence              |
| ERR_GROUP_NOT_FOUND       | Group with the specified GUID not found        | Check if the group exists                         |
| ERR_ALREADY_JOINED        | User is already a member of the group          | Check `hasJoined` before attempting to join       |
| ERR_WRONG_PASSWORD        | Incorrect password for password-protected group| Verify the password is correct                    |
| ERR_GROUP_TYPE_MISMATCH   | Provided group type doesn't match actual type  | Fetch group details first to get correct type     |
| ERR_PRIVATE_GROUP         | Cannot join private group without invitation   | User must be invited to join private groups       |

## Edge Cases & Pitfalls

<Warning>
**Group Type Mismatch**: The `groupType` parameter must match the actual type of the group. If you pass `PUBLIC` for a password-protected group, the join will fail. Always fetch group details first if you're unsure of the group type.
</Warning>

<Warning>
**Duplicate Join Attempts**: Attempting to join a group you've already joined will result in an error. Use the `hasJoined` property on the Group object to check membership status before calling `joinGroup()`.
</Warning>

<Warning>
**Private Group Restrictions**: Private groups cannot be joined directly using `joinGroup()`. Users must be invited by an existing member. Attempting to join a private group without an invitation will fail.
</Warning>

<Warning>
**Duplicate Listener IDs**: If you register a GroupListener with the same `listenerId` as an existing listener, the new listener will replace the old one. Use unique listener IDs (e.g., include timestamps or UUIDs) to avoid accidentally overwriting listeners.
</Warning>

## Performance & Scale Notes

- CometChat automatically tracks group membership. You don't need to call `joinGroup()` every time you want to communicate in a group you've already joined.
- When building group discovery features, consider paginating group lists and checking `hasJoined` to show appropriate UI (Join button vs. Open button).
- Real-time join events are delivered to all group members, so large groups may generate significant event traffic.
- For applications with many groups, consider caching group membership status locally to reduce API calls.

## Security Notes

<Note>
**Auth Key** is acceptable for POC/development only. For production, use **Auth Token** created server-side via the CometChat REST API.
</Note>

<Warning>
**Password Security**: For password-protected groups, never hardcode passwords in client-side code. Implement a secure flow where users enter the password through your UI.
</Warning>

<Warning>
**Private Group Access Control**: Private groups provide the highest level of access control. Use them for sensitive conversations where membership should be strictly controlled by existing members.
</Warning>
