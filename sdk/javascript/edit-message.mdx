## Summary

The CometChat JavaScript SDK allows you to edit previously sent text and custom messages. This page covers how to edit messages, receive real-time edit events, and handle missed edit events when your app was offline.

## When to Use This

- When allowing users to correct typos or mistakes in sent messages
- When implementing message editing functionality in your chat UI
- When updating message tags or metadata after a message has been sent
- When building moderation features that allow admins to edit messages
- When syncing edited messages across multiple devices or sessions

## Prerequisites

- **SDK**: `@cometchat/chat-sdk-javascript` v4.x
- **State**: User must be logged in via `CometChat.login()` before editing messages
- **Permissions**: User must be the message sender, or a group owner/moderator for group messages
- **Message**: The message to edit must exist and have a valid message ID

## Minimal Example

<Tabs>
<Tab title="JavaScript">
```javascript
const receiverID = "UID";
const messageText = "Updated message text";
const receiverType = CometChat.RECEIVER_TYPE.USER;
const messageId = MESSAGE_ID;

const textMessage = new CometChat.TextMessage(
  receiverID,
  messageText,
  receiverType
);
textMessage.setId(messageId);

CometChat.editMessage(textMessage).then(
  (message) => {
    console.log("Message edited successfully:", message);
  },
  (error) => {
    console.log("Message editing failed:", error);
  }
);
```

## Step-by-Step

### Edit a Message

*In other words, as a sender, how do I edit a message?*

To edit a message, use the `editMessage()` method. This method takes an object of the `BaseMessage` class. Currently, you can only edit `TextMessage` and `CustomMessage` types.

#### Add/Update Tags

While editing a message, you can update the tags associated with the message using the `setTags()` method. The tags added while editing will replace any tags set when the message was originally sent.

<Tabs>
<Tab title="JavaScript">
```javascript
const tags = ["pinnedMessage", "important"];

textMessage.setTags(tags);
```
</Tab>
<Tab title="TypeScript">
```typescript
const tags: Array<string> = ["pinnedMessage", "important"];

textMessage.setTags(tags);
```
</Tab>
</Tabs>

#### Complete Edit Example

Once the message object is ready with the updated text and/or tags, use the `editMessage()` method:

<Tabs>
<Tab title="JavaScript">
```javascript
const receiverID = "UID";
const messageText = "Hello world! (edited)";
const receiverType = CometChat.RECEIVER_TYPE.USER;
const messageId = MESSAGE_ID;

const textMessage = new CometChat.TextMessage(
  receiverID,
  messageText,
  receiverType
);
textMessage.setId(messageId);

CometChat.editMessage(textMessage).then(
  (message) => {
    console.log("Message edited:", message);
    console.log("Edited at:", message.getEditedAt());
    console.log("Edited by:", message.getEditedBy());
  },
  (error) => {
    console.log("Message editing failed with error:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript
const receiverID: string = "UID";
const messageText: string = "Hello world! (edited)";
const receiverType: string = CometChat.RECEIVER_TYPE.USER;
const messageId: number = MESSAGE_ID;

const textMessage: CometChat.TextMessage = new CometChat.TextMessage(
  receiverID,
  messageText,
  receiverType
);
textMessage.setId(messageId);

CometChat.editMessage(textMessage).then(
  (message: CometChat.TextMessage) => {
    console.log("Message edited:", message);
    console.log("Edited at:", message.getEditedAt());
    console.log("Edited by:", message.getEditedBy());
  },
  (error: CometChat.CometChatException) => {
    console.log("Message editing failed with error:", error);
  }
);
```
</Tab>
</Tabs>

The edited message object returned will contain:
- `editedAt` - Timestamp of when the message was edited
- `editedBy` - UID of the user who edited the message

### Edit Permissions

By default, CometChat allows certain roles to edit messages:

| User Role       | Conversation Type       | Edit Capabilities          |
| --------------- | ----------------------- | -------------------------- |
| Message Sender  | One-on-One Conversation | Messages they have sent    |
| Message Sender  | Group Conversation      | Messages they have sent    |
| Group Owner     | Group Conversation      | All messages in the group  |
| Group Moderator | Group Conversation      | All messages in the group  |

### Real-time Message Edit Events

*In other words, as a recipient, how do I know when someone has edited their message when my app is running?*

To receive real-time events for message edits, override the `onMessageEdited()` method of the `MessageListener` class.

#### Registering the Listener

<Tabs>
<Tab title="JavaScript">
```javascript
const listenerID = "UNIQUE_LISTENER_ID";

CometChat.addMessageListener(
  listenerID,
  new CometChat.MessageListener({
    onMessageEdited: (message) => {
      console.log("Message edited:", message);
      console.log("Edited at:", message.getEditedAt());
      console.log("Edited by:", message.getEditedBy());
    }
  })
);
```
</Tab>
<Tab title="TypeScript">
```typescript
const listenerID: string = "UNIQUE_LISTENER_ID";

CometChat.addMessageListener(
  listenerID,
  new CometChat.MessageListener({
    onMessageEdited: (message: CometChat.BaseMessage) => {
      console.log("Message edited:", message);
      console.log("Edited at:", message.getEditedAt());
      console.log("Edited by:", message.getEditedBy());
    }
  })
);
```
</Tab>
</Tabs>

#### Removing the Listener

Always remove listeners when they are no longer needed to prevent memory leaks:

<Tabs>
<Tab title="JavaScript">
```javascript
const listenerID = "UNIQUE_LISTENER_ID";

CometChat.removeMessageListener(listenerID);
```
</Tab>
<Tab title="TypeScript">
```typescript
const listenerID: string = "UNIQUE_LISTENER_ID";

CometChat.removeMessageListener(listenerID);
```
</Tab>
</Tabs>

### Missed Message Edit Events

*In other words, as a recipient, how do I know when someone edited their message when my app was not running?*

When you retrieve the list of previous messages, any edited messages will have the `editedAt` and `editedBy` fields set. Additionally, an `Action` message is created to inform you about the edit.

For example, if a conversation has 100 messages and message ID 50 was edited while your app was offline:
- Message 50 will have `editedAt` and `editedBy` fields populated
- Message 101 will be an `Action` message indicating the edit

#### Action Object Fields

For message edit events, the `Action` object contains:

| Field       | Value                                                    |
| ----------- | -------------------------------------------------------- |
| `action`    | `edited`                                                 |
| `actionOn`  | Updated message object with edited details               |
| `actionBy`  | User object of the user who edited the message           |
| `actionFor` | User/Group object of the conversation receiver           |

## API Details

### Method Signature

```
CometChat.editMessage(message: BaseMessage): Promise<BaseMessage>
```

### Parameters

| Parameter | Type        | Required | Description                                      |
| --------- | ----------- | -------- | ------------------------------------------------ |
| message   | BaseMessage | Yes      | The message object with updated content and ID   |

### Message Object Requirements

| Property    | Type   | Required | Description                                |
| ----------- | ------ | -------- | ------------------------------------------ |
| id          | number | Yes      | The ID of the message to edit (via setId)  |
| receiverID  | string | Yes      | UID or GUID of the original receiver       |
| receiverType| string | Yes      | USER or GROUP                              |
| text        | string | Yes      | The updated message text (for TextMessage) |

### Return Value

Returns a `Promise<BaseMessage>` that resolves to the edited message object containing:
- All original message properties
- `editedAt` - Timestamp of the edit
- `editedBy` - UID of the user who performed the edit

### Errors

| Error Code                  | Description                                    | Resolution                                      |
| --------------------------- | ---------------------------------------------- | ----------------------------------------------- |
| ERR_MESSAGE_NOT_FOUND       | Message with specified ID does not exist       | Verify the message ID is correct                |
| ERR_PERMISSION_DENIED       | User lacks permission to edit this message     | Ensure user is sender or group admin/moderator  |
| ERR_INVALID_MESSAGE_TYPE    | Attempting to edit unsupported message type    | Only TextMessage and CustomMessage can be edited|

## Edge Cases & Pitfalls

<Warning>
**Message ID Required**: You must call `setId()` on the message object before calling `editMessage()`. Without a valid message ID, the edit will fail. Always retrieve the original message ID from the message object you want to edit.
</Warning>

<Warning>
**Listener ID Uniqueness**: Each listener must have a unique `listenerID`. If you register multiple listeners with the same ID, only the last one will be active. Use descriptive, unique IDs like `"chat-screen-edit-listener"` to avoid conflicts.
</Warning>

<Warning>
**Memory Leaks from Listeners**: Always remove message listeners when the component unmounts or when navigating away from the chat screen. Failing to remove listeners can cause memory leaks and duplicate event handling.
</Warning>

<Warning>
**Edit Permissions**: Only the message sender can edit their own messages in one-on-one conversations. In groups, owners and moderators can edit any message. Attempting to edit without proper permissions will result in an error.
</Warning>

<Note>
In order to edit a message, you need to be either the sender of the message or the admin/moderator of the group in which the message was sent.
</Note>
