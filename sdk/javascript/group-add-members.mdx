title: "Add Members To A Group"
---

## Summary

The CometChat JavaScript SDK enables you to add members to a group programmatically. You can add one or multiple users to a group with specific roles (admin, moderator, or participant), and optionally ban users from the group in the same operation. This is essential for building group management features, invite systems, and admin tools.

## When to Use This

- When implementing an "Add Members" feature in group settings
- When building an invite system that adds users to groups
- When creating admin tools for bulk member management
- When automatically adding users to groups based on business logic
- When migrating users from one group to another
- When implementing team or organization group membership

## Prerequisites

- **SDK**: `@cometchat/chat-sdk-javascript` v4.x
- **State**: User must be logged in via `CometChat.login()`
- **Dashboard**: Ensure your CometChat app is created in the [CometChat Dashboard](https://app.cometchat.com)
- **Group**: The group must exist and the logged-in user must have permission to add members (admin or moderator)
- **Users**: The users being added must exist in your CometChat app

<Tabs>
<Tab title="JavaScript">
```javascript
let GUID = "GUID";
let UID = "UID";
let membersList = [
  new CometChat.GroupMember(UID, CometChat.GROUP_MEMBER_SCOPE.PARTICIPANT),
```

## Step-by-Step

### Step 1: Create GroupMember Objects

To add members to a group, you first need to create `GroupMember` objects for each user you want to add. Each `GroupMember` requires a UID and a scope (role).

<Tabs>
<Tab title="JavaScript">
```javascript
let UID = "UID";

// Create a participant member
let participantMember = new CometChat.GroupMember(
  UID,
  CometChat.GROUP_MEMBER_SCOPE.PARTICIPANT
);

// Create an admin member
let adminMember = new CometChat.GroupMember(
  "admin-uid",
  CometChat.GROUP_MEMBER_SCOPE.ADMIN
);

// Create a moderator member
let moderatorMember = new CometChat.GroupMember(
  "moderator-uid",
  CometChat.GROUP_MEMBER_SCOPE.MODERATOR
);
```
</Tab>

### Step 3: Add Members to the Group

Use the `addMembersToGroup()` method to add the members. The method takes the group GUID, the list of members to add, and an optional list of UIDs to ban.

<Tabs>
<Tab title="JavaScript">
```javascript
let GUID = "GUID";
let membersList = [
  new CometChat.GroupMember("user1", CometChat.GROUP_MEMBER_SCOPE.PARTICIPANT),
  new CometChat.GroupMember("user2", CometChat.GROUP_MEMBER_SCOPE.PARTICIPANT),
];
let bannedMembers = []; // Optional: UIDs to ban

CometChat.addMembersToGroup(GUID, membersList, bannedMembers).then(
  (response) => {
    console.log("Members added successfully:", response);
    // response contains status for each UID
  },
  (error) => {
    console.log("Failed to add members:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">

### Step 4: Handle the Response

The response is an object where each key is a UID and the value is either `"success"` or an error message describing why the operation failed for that user.

<Tabs>
<Tab title="JavaScript">
```javascript
let GUID = "GUID";
let membersList = [
  new CometChat.GroupMember("user1", CometChat.GROUP_MEMBER_SCOPE.PARTICIPANT),
  new CometChat.GroupMember("user2", CometChat.GROUP_MEMBER_SCOPE.PARTICIPANT),
];

CometChat.addMembersToGroup(GUID, membersList, []).then(
  (response) => {
    // Check status for each user
    for (let uid in response) {
      if (response[uid] === "success") {
        console.log(`${uid} added successfully`);
      } else {
        console.log(`Failed to add ${uid}: ${response[uid]}`);
      }
    }
  },
  (error) => {
    console.log("Failed to add members:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript

];

CometChat.addMembersToGroup(GUID, membersList, []).then(
  (response: { [key: string]: string }) => {

    console.log("Failed to add members:", error);
```

## Events & Real-time Behavior

### Registering GroupListener

To receive real-time events when members are added to a group, register a `GroupListener`. The `onMemberAddedToGroup()` callback is triggered when any user is added to a group you're a member of.

<Tabs>
<Tab title="JavaScript">
```javascript
let listenerId = "unique-group-member-listener";

CometChat.addGroupListener(
  listenerId,
  new CometChat.GroupListener({
    onMemberAddedToGroup: (message, userAdded, userAddedBy, userAddedIn) => {
      console.log("Member added to group:", {
        message,
        userAdded,
        userAddedBy,
        userAddedIn,
      });
    },
  })
);
```
</Tab>
<Tab title="TypeScript">
```typescript
let listenerId: string = "unique-group-member-listener";

CometChat.addGroupListener(
  listenerId,
  new CometChat.GroupListener({
    onMemberAddedToGroup: (
      message: CometChat.Action,
      userAdded: CometChat.User,
      userAddedBy: CometChat.User,
      userAddedIn: CometChat.Group
    ) => {
      console.log("Member added to group:", {
        message,
        userAdded,
        userAddedBy,
        userAddedIn,
      });
    },
  })
);
```
</Tab>
</Tabs>

### Removing GroupListener

Always remove listeners when they are no longer needed to prevent memory leaks and duplicate event handling.

### Complete Listener Lifecycle Example

<Tabs>
<Tab title="JavaScript">
```javascript
// Use a unique listener ID
let listenerId = "group-member-listener-" + Date.now();

// Add the listener
CometChat.addGroupListener(
  listenerId,
  new CometChat.GroupListener({
    onMemberAddedToGroup: (message, userAdded, userAddedBy, userAddedIn) => {
      console.log(`${userAdded.getName()} was added to ${userAddedIn.getName()} by ${userAddedBy.getName()}`);
    },
  })
);

// Later, when cleaning up (e.g., component unmount)
function cleanup() {
  CometChat.removeGroupListener(listenerId);
}
```

<Note>
When a group member is added by another member, the `onMemberAddedToGroup` event is triggered. When a user joins a group on their own (via `joinGroup()`), the `onGroupMemberJoined` event is triggered instead.
</Note>

### Member Added Events in Message History

When retrieving previous messages, if a member was added to a group, the message list will contain an `Action` message. The `Action` object provides the following information:

| Property    | Description                                                |
| ----------- | ---------------------------------------------------------- |
| action      | `"added"` - indicates a member was added                   |
| actionOn    | User object of the user who was added to the group         |
| actionBy    | User object of the user who added the member               |
| actionFor   | Group object of the group to which the member was added    |

## API Details

### Method Signature

```javascript
CometChat.addMembersToGroup(
  GUID: string,
  membersList: GroupMember[],
  bannedMembers: string[]
): Promise<Object>
```

### Parameters

| Parameter     | Type           | Required | Description                                                    |
| ------------- | -------------- | -------- | -------------------------------------------------------------- |
| GUID          | string         | Yes      | The unique identifier of the group                             |
| membersList   | GroupMember[]  | Yes      | Array of GroupMember objects to add to the group               |
| bannedMembers | string[]       | No       | Array of UIDs to ban from the group (can be empty array)       |

### GroupMember Constructor

| Parameter | Type   | Required | Description                                                    |
| --------- | ------ | -------- | -------------------------------------------------------------- |
| UID       | string | Yes      | The unique identifier of the user to add                       |
| scope     | string | Yes      | The role of the member (ADMIN, MODERATOR, or PARTICIPANT)      |

### Scope Constants

| Constant                                  | Value         | Description                              |
| ----------------------------------------- | ------------- | ---------------------------------------- |
| CometChat.GROUP_MEMBER_SCOPE.ADMIN        | "admin"       | Full group management permissions        |
| CometChat.GROUP_MEMBER_SCOPE.MODERATOR    | "moderator"   | Can manage members and messages          |
| CometChat.GROUP_MEMBER_SCOPE.PARTICIPANT  | "participant" | Standard member with basic permissions   |

### Return Value

Returns a `Promise<Object>` that resolves to an object where:
- Each key is a UID from the membersList or bannedMembers
- Each value is either `"success"` or an error message string

### Errors

| Error Code                  | Description                                    | Resolution                                      |
| --------------------------- | ---------------------------------------------- | ----------------------------------------------- |
| ERR_USER_NOT_LOGGED_IN      | No user is currently logged in                 | Ensure user is logged in before adding members  |
| ERR_INVALID_GUID            | The provided GUID is invalid                   | Verify the GUID format and existence            |
| ERR_GROUP_NOT_FOUND         | Group with the specified GUID not found        | Check if the group exists                       |
| ERR_NOT_A_MEMBER            | User is not a member of the group              | User must be a member to add others             |
| ERR_PERMISSION_DENIED       | User doesn't have permission to add members    | Only admins and moderators can add members      |
| ERR_USER_NOT_FOUND          | One or more UIDs in the list don't exist       | Verify all UIDs exist in your CometChat app     |
| ERR_ALREADY_MEMBER          | User is already a member of the group          | Check membership before adding                  |

## Edge Cases & Pitfalls

<Warning>
**Permission Requirements**: Only group admins and moderators can add members to a group. Participants cannot add members. If a participant attempts to add members, the operation will fail with a permission error. Always verify the logged-in user's scope before showing the "Add Members" UI.
</Warning>

<Warning>
**Partial Success Handling**: When adding multiple members, some may succeed while others fail (e.g., if a user is already a member or doesn't exist). Always iterate through the response object to check the status of each UID rather than assuming all operations succeeded.
</Warning>

<Warning>
**Duplicate Listener IDs**: If you register a GroupListener with the same `listenerId` as an existing listener, the new listener will replace the old one. Use unique listener IDs (e.g., include timestamps or UUIDs) to avoid accidentally overwriting listeners, especially in applications with multiple components listening for group events.
</Warning>

<Warning>
**Banned User Conflicts**: If you try to add a user who is currently banned from the group, the operation will fail for that user. You must unban the user first using `unbanGroupMember()` before adding them back to the group.
</Warning>

## Performance & Scale Notes

- You can add multiple members in a single API call, which is more efficient than making individual calls for each member.
- The maximum number of members that can be added in a single call depends on your CometChat plan. Check your plan limits in the Dashboard.
- For large-scale member additions (e.g., migrating hundreds of users), consider batching the operations and implementing retry logic for failed additions.
- The response object size scales with the number of members being added, so parse it efficiently for large batches.

## Security Notes

<Note>
**Auth Key** is acceptable for POC/development only. For production, use **Auth Token** created server-side via the CometChat REST API.
</Note>

<Warning>
**Server-Side Validation**: For production applications, consider implementing server-side validation before adding members. This allows you to enforce business rules (e.g., subscription limits, organization membership) before the CometChat API is called.
</Warning>

<Warning>
**Scope Assignment**: Be careful when assigning admin or moderator scopes. These roles have elevated permissions including the ability to kick, ban, and manage other members. Only assign elevated scopes to trusted users.
</Warning>
