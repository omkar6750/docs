## Summary

The CometChat JavaScript SDK enables threaded conversations by allowing messages to be sent as replies to a parent message. This page covers how to send messages in threads, receive real-time thread messages, fetch thread history, and filter out threaded messages from main conversations.

## When to Use This

- When implementing reply-to-message functionality in your chat UI
- When building discussion threads within a conversation
- When organizing related messages under a parent message
- When allowing users to have focused sub-conversations without cluttering the main chat
- When building features like comment threads on specific messages

## Prerequisites

- **SDK**: `@cometchat/chat-sdk-javascript` v4.x
- **State**: User must be logged in via `CometChat.login()` before sending or fetching threaded messages
- **Parent Message**: A valid parent message ID is required to create or fetch thread messages
- **Conversation**: An existing user or group conversation where the parent message exists

## Minimal Example

<Tabs>
<Tab title="JavaScript">
```javascript
const receiverID = "UID";
const messageText = "This is a reply in the thread";
const receiverType = CometChat.RECEIVER_TYPE.USER;
const parentMessageId = MESSAGE_ID;

const textMessage = new CometChat.TextMessage(receiverID, messageText, receiverType);
textMessage.setParentMessageId(parentMessageId);

CometChat.sendMessage(textMessage).then(
  (message) => {
    console.log("Thread message sent successfully:", message);
  },
  (error) => {
    console.log("Thread message sending failed:", error);
  }
);
```

## Step-by-Step

### Send a Message in a Thread

*In other words, how do I reply to a specific message and create a thread?*

Messages that are started from a particular message are called threaded messages or simply threads. Each thread is attached to a message which is the parent message for that thread.

To send a message in a thread, use the `setParentMessageId()` method on your message object before sending. This method is available on `TextMessage`, `MediaMessage`, and `CustomMessage` classes.

#### Send Text Message in a Thread (User Conversation)

<Tabs>
<Tab title="JavaScript">
```javascript
const receiverID = "UID";
const parentMessageId = MESSAGE_ID;

const textMessage = new CometChat.TextMessage(
  receiverID,
  "Hello, this is a thread reply!",
  CometChat.RECEIVER_TYPE.USER
);
textMessage.setParentMessageId(parentMessageId);

CometChat.sendMessage(textMessage).then(
  (message) => {
    console.log("Message sent successfully:", message);
  },
  (error) => {
    console.log("Message sending failed:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">

### Receive Real-Time Thread Messages

*In other words, how do I know when someone sends a message in a thread I'm viewing?*

The procedure to receive real-time messages is exactly the same as receiving regular messages. Use the `MessageListener` class to listen for incoming messages and check the `parentMessageId` field to determine if the message belongs to the active thread.

#### Registering the Listener

<Tabs>
<Tab title="JavaScript">
```javascript
const listenerID = "UNIQUE_LISTENER_ID";
const activeThreadId = MESSAGE_ID;

CometChat.addMessageListener(
  listenerID,
  new CometChat.MessageListener({
    onTextMessageReceived: (textMessage) => {
      if (textMessage.getParentMessageId() === activeThreadId) {
        console.log("Text message received for active thread:", textMessage);
      }
    },
    onMediaMessageReceived: (mediaMessage) => {
      if (mediaMessage.getParentMessageId() === activeThreadId) {
        console.log("Media message received for active thread:", mediaMessage);
      }
    },
    onCustomMessageReceived: (customMessage) => {
      if (customMessage.getParentMessageId() === activeThreadId) {
        console.log("Custom message received for active thread:", customMessage);
      }
    }
  })
);
```
</Tab>
<Tab title="TypeScript">
```typescript
const listenerID: string = "UNIQUE_LISTENER_ID";
const activeThreadId: number = MESSAGE_ID;

CometChat.addMessageListener(
  listenerID,
  new CometChat.MessageListener({

      }

  })
);
```
</Tab>
</Tabs>

### Fetch Messages for a Thread

*In other words, how do I load the message history for a specific thread?*

Use the `MessagesRequestBuilder` class with the `setParentMessageId()` method to fetch all messages belonging to a particular thread.

<Tabs>
<Tab title="JavaScript">
```javascript
const limit = 30;
const parentMessageId = MESSAGE_ID;

const messagesRequest = new CometChat.MessagesRequestBuilder()
  .setLimit(limit)
  .setParentMessageId(parentMessageId)
  .build();

messagesRequest.fetchPrevious().then(
  (messages) => {
    console.log("Thread messages fetched successfully:", messages);
  },
  (error) => {
    console.log("Message fetching failed:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript
const limit: number = 30;
const parentMessageId: number = MESSAGE_ID;

const messagesRequest: CometChat.MessagesRequest = new CometChat.MessagesRequestBuilder()
  .setLimit(limit)
  .setParentMessageId(parentMessageId)
  .build();

messagesRequest.fetchPrevious().then(
  (messages: CometChat.BaseMessage[]) => {
    console.log("Thread messages fetched successfully:", messages);
  },
  (error: CometChat.CometChatException) => {
    console.log("Message fetching failed:", error);
  }
);
```
</Tab>
</Tabs>

In one request, a maximum of 100 messages can be fetched. To fetch the next set of messages, call `fetchPrevious()` again on the same `MessagesRequest` object.

### Exclude Threaded Messages from Main Conversation

*In other words, how do I hide thread replies from the main message list?*

By default, threaded messages are included when fetching messages for a user/group conversation. To exclude them, use the `hideReplies()` method of the `MessagesRequestBuilder` class.

#### User Conversation

<Tabs>
<Tab title="JavaScript">
```javascript
const UID = "UID";
const limit = 30;

const messagesRequest = new CometChat.MessagesRequestBuilder()
  .setUID(UID)
  .setLimit(limit)
  .hideReplies(true)
  .build();

messagesRequest.fetchPrevious().then(
  (messages) => {
    console.log("Messages fetched (excluding thread replies):", messages);
  },
  (error) => {
    console.log("Message fetching failed:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript
const UID: string = "UID";
const limit: number = 30;

  .setUID(UID)

    console.log("Message fetching failed:", error);
  }
);
```

## API Details

### setParentMessageId Method

Sets the parent message ID to associate a message with a thread.

```
message.setParentMessageId(parentMessageId: number): void
```

#### Parameters

| Parameter       | Type   | Required | Description                                      |
| --------------- | ------ | -------- | ------------------------------------------------ |
| parentMessageId | number | Yes      | The ID of the parent message to create a thread  |

### getParentMessageId Method

Retrieves the parent message ID from a message object.

```
message.getParentMessageId(): number
```

#### Return Value

Returns the parent message ID as a `number`. Returns `0` or `undefined` if the message is not part of a thread.

### MessagesRequestBuilder.setParentMessageId

Configures the request to fetch messages for a specific thread.

```
new CometChat.MessagesRequestBuilder().setParentMessageId(parentMessageId: number): MessagesRequestBuilder
```

#### Parameters

| Parameter       | Type   | Required | Description                                    |
| --------------- | ------ | -------- | ---------------------------------------------- |
| parentMessageId | number | Yes      | The ID of the parent message for the thread    |

### MessagesRequestBuilder.hideReplies

Configures the request to exclude threaded messages from results.

```
new CometChat.MessagesRequestBuilder().hideReplies(hideReplies: boolean): MessagesRequestBuilder
```

#### Parameters

| Parameter   | Type    | Required | Description                                           |
| ----------- | ------- | -------- | ----------------------------------------------------- |
| hideReplies | boolean | Yes      | Set to `true` to exclude thread replies from results  |

### Errors

| Error Code            | Description                                    | Resolution                                       |
| --------------------- | ---------------------------------------------- | ------------------------------------------------ |
| ERR_NOT_LOGGED_IN     | User is not logged in                          | Call CometChat.login() before sending messages   |
| ERR_INVALID_MESSAGE   | Invalid message object or missing parameters   | Ensure message object is properly constructed    |
| ERR_MESSAGE_NOT_FOUND | Parent message with specified ID not found     | Verify the parent message ID exists              |

## Edge Cases & Pitfalls

<Warning>
**Parent Message Must Exist**: The `parentMessageId` must reference an existing message. If you pass an invalid or non-existent message ID, the message will still be sent but won't be properly associated with a thread. Always verify the parent message exists before creating thread replies.
</Warning>

<Warning>
**Thread Messages in Main List**: By default, thread replies appear in the main conversation message list. If you want a cleaner main conversation view, always use `hideReplies(true)` when fetching messages for the main conversation, and fetch thread messages separately using `setParentMessageId()`.
</Warning>

<Warning>
**Listener ID Uniqueness**: Each listener must have a unique `listenerID`. If you register multiple listeners with the same ID, only the last one will be active. Use descriptive, unique IDs like `"thread-view-listener"` to avoid conflicts.
</Warning>

<Warning>
**Memory Leaks from Listeners**: Always remove message listeners when navigating away from the thread view or when the component unmounts. Failing to remove listeners can cause memory leaks and duplicate event handling.
</Warning>

<Warning>
**Pagination for Large Threads**: Threads can contain many messages. Always implement pagination by calling `fetchPrevious()` multiple times on the same `MessagesRequest` object. Each call returns up to 100 messages.
</Warning>
