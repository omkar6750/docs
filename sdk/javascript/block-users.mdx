## Summary

The CometChat JavaScript SDK allows users to block and unblock other users, preventing all communication between them. This page covers blocking users, unblocking users, and retrieving the list of blocked users. Blocking is essential for user safety and moderation in chat applications.

## When to Use This

- When a user wants to stop receiving messages from another user
- When implementing user safety features in your chat application
- When building moderation tools to manage user interactions
- When you need to retrieve a list of users that have been blocked
- When allowing users to unblock previously blocked users

## Prerequisites

- **SDK**: `@cometchat/chat-sdk-javascript` v4.x
- **State**: User must be logged in via `CometChat.login()`
- **Dashboard**: Ensure your CometChat app is created in the [CometChat Dashboard](https://app.cometchat.com)

## Minimal Example

<Tabs>
<Tab title="JavaScript">
```javascript
let usersList = ["UID1", "UID2", "UID3"];

CometChat.blockUsers(usersList).then(
  (list) => {

  },

    console.log("Blocking users failed:", error);

```
</Tab>


## Step-by-Step

### Step 1: Block Users

You can block users using the `blockUsers()` method. Once a user is blocked, all communication to and from that user will be completely blocked. You can block multiple users in a single operation.

The `blockUsers()` method takes an `Array` of UIDs as a parameter.

<Tabs>
<Tab title="JavaScript">
```javascript
let usersList = ["UID1", "UID2", "UID3"];

CometChat.blockUsers(usersList).then(
  (list) => {
    console.log("Users list blocked:", list);
  },
  (error) => {
    console.log("Blocking user fails with error:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript
let usersList: string[] = ["UID1", "UID2", "UID3"];

CometChat.blockUsers(usersList).then(
  (list: Object) => {
    console.log("Users list blocked:", list);
  },
  (error: CometChat.CometChatException) => {
    console.log("Blocking user fails with error:", error);
  }
);
```
</Tab>
</Tabs>

The method returns an Object with UIDs as keys and `"success"` or `"fail"` as values, indicating whether the block operation succeeded for each UID.

### Step 2: Unblock Users

You can unblock previously blocked users using the `unblockUsers()` method. You can unblock multiple users in a single operation.

<Tabs>
<Tab title="JavaScript">
```javascript
let usersList = ["UID1", "UID2", "UID3"];

CometChat.unblockUsers(usersList).then(
  (list) => {
    console.log("Users list unblocked:", list);
  },
  (error) => {
    console.log("Unblocking user fails with error:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript
let usersList: string[] = ["UID1", "UID2", "UID3"];

CometChat.unblockUsers(usersList).then(
  (list: Object) => {
    console.log("Users list unblocked:", list);
  },
  (error: CometChat.CometChatException) => {
    console.log("Unblocking user fails with error:", error);
  }
);
```
</Tab>
</Tabs>

The method returns an Object with UIDs as keys and `"success"` or `"fail"` as values, indicating whether the unblock operation succeeded for each UID.

### Step 3: Retrieve Blocked Users

To fetch the list of blocked users, use the `BlockedUsersRequest` class. Create an instance using `BlockedUsersRequestBuilder` to configure the request parameters.

#### Set Limit

Set the maximum number of blocked users to fetch in a single request:

<Tabs>
<Tab title="JavaScript">
```javascript
let limit = 30;
let blockedUsersRequest = new CometChat.BlockedUsersRequestBuilder()
  .setLimit(limit)
  .build();
```
</Tab>
<Tab title="TypeScript">
```typescript
let limit: number = 30;
let blockedUsersRequest: CometChat.BlockedUsersRequest = new CometChat.BlockedUsersRequestBuilder()
  .setLimit(limit)
  .build();
```
</Tab>
</Tabs>

#### Set Search Keyword

Filter blocked users by a search string:

<Tabs>
<Tab title="JavaScript">
```javascript
let limit = 30;
let searchKeyword = "super";
let blockedUsersRequest = new CometChat.BlockedUsersRequestBuilder()
  .setLimit(limit)
  .setSearchKeyword(searchKeyword)
  .build();
```
</Tab>
<Tab title="TypeScript">
```typescript
let limit: number = 30;
let searchKeyword: string = "super";
let blockedUsersRequest: CometChat.BlockedUsersRequest = new CometChat.BlockedUsersRequestBuilder()
  .setLimit(limit)
  .setSearchKeyword(searchKeyword)
  .build();
```
</Tab>
</Tabs>

#### Set Direction

Filter by block direction:

- `CometChat.BlockedUsersRequest.directions.BLOCKED_BY_ME` - Users blocked by the logged-in user
- `CometChat.BlockedUsersRequest.directions.HAS_BLOCKED_ME` - Users who have blocked the logged-in user
- `CometChat.BlockedUsersRequest.directions.BOTH` - Both cases (default)

<Tabs>
<Tab title="JavaScript">
```javascript
let limit = 30;
let blockedUsersRequest = new CometChat.BlockedUsersRequestBuilder()
  .setLimit(limit)
  .setDirection(CometChat.BlockedUsersRequest.directions.BLOCKED_BY_ME)
  .build();
```
</Tab>
<Tab title="TypeScript">
```typescript
let limit: number = 30;
let blockedUsersRequest: CometChat.BlockedUsersRequest = new CometChat.BlockedUsersRequestBuilder()
  .setLimit(limit)
  .setDirection(CometChat.BlockedUsersRequest.directions.BLOCKED_BY_ME)
  .build();
```
</Tab>
</Tabs>

#### Fetch Blocked Users

Once you have configured the request, call `fetchNext()` to retrieve the blocked users:

<Tabs>
<Tab title="JavaScript">
```javascript
let limit = 30;
let blockedUsersRequest = new CometChat.BlockedUsersRequestBuilder()
  .setLimit(limit)
  .build();

blockedUsersRequest.fetchNext().then(
  (userList) => {
    console.log("Blocked user list received:", userList);
  },
  (error) => {
    console.log("Blocked user list fetching failed with error:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript
let limit: number = 30;
let blockedUsersRequest: CometChat.BlockedUsersRequest = new CometChat.BlockedUsersRequestBuilder()
  .setLimit(limit)
  .build();

blockedUsersRequest.fetchNext().then(
  (userList: CometChat.User[]) => {
    console.log("Blocked user list received:", userList);
  },
  (error: CometChat.CometChatException) => {
    console.log("Blocked user list fetching failed with error:", error);
  }
);
```
</Tab>
</Tabs>

## API Details

### Method Signatures

```javascript
CometChat.blockUsers(usersList: string[]): Promise<Object>
CometChat.unblockUsers(usersList: string[]): Promise<Object>
BlockedUsersRequest.fetchNext(): Promise<User[]>
```

### Parameters

#### blockUsers / unblockUsers

| Parameter  | Type     | Required | Description                              |
| ---------- | -------- | -------- | ---------------------------------------- |
| usersList  | string[] | Yes      | Array of UIDs to block or unblock        |

#### BlockedUsersRequestBuilder

| Method            | Parameter Type | Description                                           |
| ----------------- | -------------- | ----------------------------------------------------- |
| setLimit          | number         | Maximum number of users to fetch per request (max 100)|
| setSearchKeyword  | string         | Filter users by name or UID                           |
| setDirection      | string         | Filter by block direction (BLOCKED_BY_ME, HAS_BLOCKED_ME, BOTH) |

### Return Values

#### blockUsers / unblockUsers

Returns a `Promise<Object>` where:
- Keys are the UIDs from the input array
- Values are `"success"` or `"fail"` indicating the operation result for each UID

#### fetchNext

Returns a `Promise<User[]>` containing an array of User objects representing blocked users.

### Errors

| Error Code                  | Description                                    | Resolution                                      |
| --------------------------- | ---------------------------------------------- | ----------------------------------------------- |
| ERR_UID_NOT_FOUND           | One or more UIDs do not exist                  | Verify UIDs exist before blocking               |
| ERR_NOT_LOGGED_IN           | User is not logged in                          | Ensure user is logged in via CometChat.login()  |
| ERR_INVALID_LIMIT           | Limit value is invalid                         | Use a positive number up to 100                 |

## Edge Cases & Pitfalls

<Warning>
**Blocking Yourself**: Attempting to block your own UID will fail. The SDK does not allow users to block themselves. Always filter out the logged-in user's UID from the block list.
</Warning>

<Warning>
**Partial Failures**: When blocking multiple users, some operations may succeed while others fail. Always check the returned object for each UID's status (`"success"` or `"fail"`) rather than assuming all operations succeeded.
</Warning>

<Warning>
**Pagination Required**: The `fetchNext()` method returns a limited number of users per call (up to the limit you set). For users with many blocked contacts, you must call `fetchNext()` repeatedly until it returns an empty array to get the complete list.
</Warning>

<Warning>
**Block Status Not Real-Time**: Blocking a user does not immediately notify them. The blocked user will only notice when they attempt to send a message and it fails. Consider updating your UI to reflect block status changes.
</Warning>

## Performance & Scale Notes

- The `blockUsers()` and `unblockUsers()` methods support batch operations—block or unblock multiple users in a single API call for better performance
- The maximum limit for `BlockedUsersRequest` is 100 users per request
- For applications with many blocked users, implement pagination by calling `fetchNext()` until an empty array is returned
- Block status is cached; changes may take a moment to propagate across all active sessions

## Security Notes

<Note>
Blocking is a user-initiated action that does not require Auth Key. The logged-in user can only block/unblock users on their own behalf—they cannot block users on behalf of other users.
</Note>

<Warning>
**Moderation Considerations**: For admin-level blocking or moderation features, use the [REST API](https://api-explorer.cometchat.com/reference/block-users) from your server. This allows administrators to manage blocks without exposing moderation capabilities to end users.
</Warning>
