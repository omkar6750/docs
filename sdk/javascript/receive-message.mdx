## Summary

The CometChat JavaScript SDK enables you to receive real-time messages when your app is running and retrieve missed messages when your app was offline. This page covers setting up message listeners, fetching missed and unread messages, retrieving message history, searching messages, and getting unread message counts.

## When to Use This

- When implementing real-time chat to receive incoming messages instantly
- When loading missed messages after the app comes back online
- When fetching unread messages to show notification badges
- When implementing message history with infinite scroll
- When building search functionality to find specific messages
- When displaying unread message counts in conversation lists

## Prerequisites

- **SDK**: `@cometchat/chat-sdk-javascript` v4.x
- **State**: User must be logged in via `CometChat.login()` before receiving messages
- **Init**: `CometChat.init()` must be called successfully before adding listeners
- **For Missed Messages**: Store the last delivered message ID to fetch messages since that point

## Step-by-Step

Receiving messages with CometChat has two parts:

1. Adding a listener to receive [real-time messages](#real-time-messages) when your app is running
2. Calling a method to retrieve [missed messages](#missed-messages) when your app was not running

## Real-Time Messages

*In other words, as a recipient, how do I receive messages when my app is running?*

To receive real-time incoming messages, you need to register the `MessageListener` wherever you wish to receive the incoming messages. You can use the `addMessageListener()` method to do so.

<Tabs>
<Tab title="JavaScript">
```javascript
const listenerId = "unique-listener-id";

CometChat.addMessageListener(
  listenerId,
  new CometChat.MessageListener({
    onTextMessageReceived: (textMessage) => {
      console.log("Text message received successfully", textMessage);
    },
    onMediaMessageReceived: (mediaMessage) => {
      console.log("Media message received successfully", mediaMessage);
    },
    onCustomMessageReceived: (customMessage) => {
      console.log("Custom message received successfully", customMessage);
    },
  })
);
```
</Tab>
<Tab title="TypeScript">
```typescript
const listenerId: string = "unique-listener-id";

CometChat.addMessageListener(
  listenerId,
  new CometChat.MessageListener({
    onTextMessageReceived: (textMessage: CometChat.TextMessage) => {
      console.log("Text message received successfully", textMessage);
    },
    onMediaMessageReceived: (mediaMessage: CometChat.MediaMessage) => {
      console.log("Media message received successfully", mediaMessage);
    },
    onCustomMessageReceived: (customMessage: CometChat.CustomMessage) => {
      console.log("Custom message received successfully", customMessage);
    },
  })
);
```
</Tab>
</Tabs>

| Parameter      | Description                                   |
| -------------- | --------------------------------------------- |
| **listenerId** | An ID that uniquely identifies that listener. |

We recommend you remove the listener once you don't want to receive messages for a particular listener.

<Tabs>
<Tab title="JavaScript">
```javascript
const listenerId = "unique-listener-id";

CometChat.removeMessageListener(listenerId);
```
</Tab>
<Tab title="TypeScript">
```typescript
const listenerId: string = "unique-listener-id";

CometChat.removeMessageListener(listenerId);
```
</Tab>
</Tabs>


<Warning>
As a sender, you will not receive your own message in a real-time message event. However, if a user is logged-in using multiple devices, they will receive an event for their own message in other devices.
</Warning>

## Missed Messages

*In other words, as a recipient, how do I receive messages that I missed when my app was not running?*

For most use cases, you will need to add functionality to load missed messages. If you're building an on-demand or live streaming app, you may choose to skip this and fetch message history instead.

Using the same `MessagesRequest` class and the filters provided by the `MessagesRequestBuilder` class, you can fetch the messages that were sent to the logged-in user but were not delivered to them as they were offline. For this, you will require the ID of the last message received. You can either maintain it at your end or use the `getLastDeliveredMessageId()` method provided by the CometChat class. This ID needs to be passed to the `setMessageId()` method of the builder class.

Now using the `fetchNext()` method, you can fetch all the messages that were sent to the user when they were offline.

Calling the `fetchNext()` method on the same object repeatedly allows you to fetch all the offline messages for the logged-in user.

### Fetch Missed Messages of a particular one-on-one conversation

<Tabs>
<Tab title="JavaScript">
```javascript
const UID = "UID";
const limit = 30;
const latestId = await CometChat.getLastDeliveredMessageId();

const messagesRequest = new CometChat.MessagesRequestBuilder()
  .setUID(UID)
  .setMessageId(latestId)
  .setLimit(limit)
  .build();

messagesRequest.fetchNext().then(
  (messages) => {
    console.log("Message list fetched:", messages);
  },
  (error) => {
    console.log("Message fetching failed with error:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript
const UID: string = "UID";
const limit: number = 30;
const latestId: number = await CometChat.getLastDeliveredMessageId();

const messagesRequest: CometChat.MessagesRequest =
  new CometChat.MessagesRequestBuilder()
    .setUID(UID)
    .setMessageId(latestId)
    .setLimit(limit)
    .build();

messagesRequest.fetchNext().then(
  (messages: CometChat.BaseMessage[]) => {
    console.log("Message list fetched:", messages);
  },
  (error: CometChat.CometChatException) => {
    console.log("Message fetching failed with error:", error);
  }
);
```
</Tab>
</Tabs>

## Search Messages

*In other words, as a logged-in user, how do I search for a message?*

In order to do this, you can use the `setSearchKeyword()` method. This method accepts a string as input. When set, the SDK will fetch messages which match the `searchKeyword`.

<Note>
By default, the searchKey is searched only in message text. However, if you enable `Conversation & Advanced Search`, the searchKey will be searched in message text, file name, mentions & mime type of the file.

The `Conversation & Advanced Search` is only available in `Advanced` & `Custom` [plans](https://www.cometchat.com/pricing). If you're already on one of these plans, please enable the `Conversation & Advanced Search` from [CometChat Dashboard](https://app.cometchat.com) (Open your app, navigate to Chats → Settings → General Configuration)
</Note>

| Feature          | Basic Search | Advanced Search |
| ---------------- | ------------ | --------------- |
| Text search      | ✅            | ✅               |
| File name search | ❌            | ✅               |
| Mentions search  | ❌            | ✅               |
| Mime type search | ❌            | ✅               |


### Search Messages in a particular one-on-one conversation

<Tabs>
<Tab title="JavaScript">
```javascript
const UID = "UID";
const limit = 30;
```

## Unread Message Count

*In other words, as a logged-in user, how do I find out the number of unread messages that I have?*

### Fetch Unread Message Count of a particular one-on-one conversation

*In other words, how do I find out the number of unread messages I have from a particular user?*

In order to get the unread message count for a particular user, you can use the `getUnreadMessageCountForUser()` method.

This method has the below two variants:

<Tabs>
<Tab title="JavaScript">
```javascript
const UID = "UID";
CometChat.getUnreadMessageCountForUser(UID);
```
</Tab>
<Tab title="TypeScript">
```typescript
const UID: string = "UID";
CometChat.getUnreadMessageCountForUser(UID);
```
</Tab>
</Tabs>

If you wish to ignore the messages from blocked users you can use the below syntax setting the boolean parameter to true:

<Tabs>
<Tab title="JavaScript">
```javascript
const UID = "UID";
const hideMessagesFromBlockedUsers = true;
CometChat.getUnreadMessageCountForUser(UID, hideMessagesFromBlockedUsers);
```
</Tab>
<Tab title="TypeScript">
```typescript
const UID: string = "UID";
const hideMessagesFromBlockedUsers: boolean = true;
CometChat.getUnreadMessageCountForUser(UID, hideMessagesFromBlockedUsers);
```
</Tab>
</Tabs>

<Tabs>
<Tab title="JavaScript">
```javascript
const UID = "UID";

CometChat.getUnreadMessageCountForUser(UID).then(
  (unreadMessageCountObject) => {
    console.log("Unread message count fetched", unreadMessageCountObject);
  },
  (error) => {
    console.log("Error in getting unread message count", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript
const UID: string = "UID";

CometChat.getUnreadMessageCountForUser(UID).then(
  (unreadMessageCount: Object) => {
    console.log("Unread message count fetched", unreadMessageCount);
  },
  (error: CometChat.CometChatException) => {
    console.log("Error in getting unread message count", error);
  }
);
```
</Tab>
</Tabs>

It will return an object which will contain the UID as the key and the unread message count as the value.

## API Details

### addMessageListener

#### Method Signature

`CometChat.addMessageListener(listenerId: string, messageListener: MessageListener): void`

#### Parameters

| Parameter       | Type            | Required | Default | Description                                      |
| --------------- | --------------- | -------- | ------- | ------------------------------------------------ |
| listenerId      | string          | Yes      | -       | Unique identifier for the listener               |
| messageListener | MessageListener | Yes      | -       | MessageListener object with callback methods     |

| Callback                  | Parameter Type        | Description                              |
| ------------------------- | --------------------- | ---------------------------------------- |
| onTextMessageReceived     | CometChat.TextMessage | Called when a text message is received   |
| onMediaMessageReceived    | CometChat.MediaMessage| Called when a media message is received  |
| onCustomMessageReceived   | CometChat.CustomMessage| Called when a custom message is received |

#### Method Signature

`CometChat.removeMessageListener(listenerId: string): void`

#### Parameters

| Parameter  | Type   | Required | Default | Description                        |
| ---------- | ------ | -------- | ------- | ---------------------------------- |
| listenerId | string | Yes      | -       | The ID of the listener to remove   |

### MessagesRequestBuilder

#### Method Signature

`new CometChat.MessagesRequestBuilder()`

#### Builder Methods

| Method            | Parameter Type | Description                                              |
| ----------------- | -------------- | -------------------------------------------------------- |
| setUID(uid)       | string         | Set user ID to fetch messages for one-on-one conversation|
| setGUID(guid)     | string         | Set group ID to fetch messages for group conversation    |
| setLimit(limit)   | number         | Set maximum number of messages to fetch (max 100)        |
| setMessageId(id)  | number         | Set message ID to fetch messages after this ID           |
| setUnread(unread) | boolean        | Set to true to fetch only unread messages                |
| setSearchKeyword(keyword) | string | Set search keyword to filter messages                    |
| build()           | -              | Build and return the MessagesRequest object              |

### MessagesRequest Methods

| Method          | Return Type                    | Description                                |
| --------------- | ------------------------------ | ------------------------------------------ |
| fetchNext()     | Promise\<CometChat.BaseMessage[]\> | Fetch next batch of messages (newer)       |
| fetchPrevious() | Promise\<CometChat.BaseMessage[]\> | Fetch previous batch of messages (older)   |

### Common Errors

| Error Code           | Description                        | Resolution                                      |
| -------------------- | ---------------------------------- | ----------------------------------------------- |
| ERR_NOT_LOGGED_IN    | User not logged in                 | Call `CometChat.login()` before fetching messages |
| ERR_UID_NOT_FOUND    | User UID doesn't exist             | Verify the user exists in CometChat             |
| ERR_GUID_NOT_FOUND   | Group GUID doesn't exist           | Verify the group exists in CometChat            |
| ERR_NOT_A_MEMBER     | User is not a member of the group  | Join the group before fetching messages         |

### Event Ordering

- Messages are delivered in the order they are received by the CometChat servers
- For missed messages, use `fetchNext()` with `setMessageId()` to get messages in chronological order
- For message history, use `fetchPrevious()` to get messages in reverse chronological order (newest first)


## Edge Cases & Pitfalls

<Warning>
**Pitfall 1: Duplicate listener registration**

Registering multiple listeners with the same `listenerId` will overwrite the previous listener. This can cause missed messages if you accidentally use the same ID. Always use unique listener IDs, especially when registering listeners in multiple components. Consider using component-specific prefixes like `"chat-screen-listener"` or `"notification-listener"`.
</Warning>

<Warning>
**Pitfall 2: Not removing listeners on component unmount**

Failing to remove message listeners when a component unmounts leads to memory leaks and duplicate event handling. In React, always use the cleanup function in `useEffect`. In Vue, use `onUnmounted`. In Angular, use `ngOnDestroy`. This is especially critical in single-page applications where components mount and unmount frequently.
</Warning>

<Warning>
**Pitfall 3: Confusing fetchNext() and fetchPrevious()**

`fetchNext()` fetches newer messages (use with `setMessageId()` for missed messages), while `fetchPrevious()` fetches older messages (use for message history with infinite scroll). Using the wrong method will return unexpected results or empty arrays.
</Warning>

<Warning>
**Pitfall 4: Not handling all message types**

The `MessageListener` has separate callbacks for text, media, and custom messages. If you only implement `onTextMessageReceived`, you'll miss media and custom messages. Always implement all relevant callbacks or use a base handler that checks the message type.
</Warning>

<Warning>
**Pitfall 5: Expecting to receive your own messages**

As a sender, you will NOT receive your own messages in real-time events. However, if the same user is logged in on multiple devices, other devices will receive the message. Design your UI to optimistically display sent messages without waiting for a listener callback.
</Warning>

## Performance & Scale Notes

- **Pagination limit**: Maximum 100 messages per request. Use pagination for large message histories.
- **Listener efficiency**: Register listeners at the highest appropriate level in your component tree to avoid duplicate registrations.
- **Unread count caching**: Cache unread counts locally and update them based on real-time events rather than fetching repeatedly.
- **Message history**: Implement infinite scroll with `fetchPrevious()` to load messages on demand rather than fetching all at once.
- **Search performance**: Advanced search features require the `Conversation & Advanced Search` add-on for optimal performance on large datasets.

## Security Notes

<Note>
**Auth Key** is acceptable for POC/development only. For production, use **Auth Token** created server-side via the CometChat REST API.
</Note>

<Note>
**Message Content**

Messages received may contain user-generated content. Always sanitize message content before rendering in your UI to prevent XSS attacks. Never execute code or scripts from message content.
</Note>

<Warning>
**Listener Security**

Message listeners receive all messages for the logged-in user. Ensure your application properly handles message visibility and doesn't expose messages to unauthorized users in multi-user scenarios.
</Warning>
