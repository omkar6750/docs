## Summary

The CometChat JavaScript SDK enables you to send text messages, media messages (images, videos, audio, files), and custom messages (like location data) to individual users or groups. This page covers all message types, metadata attachment, message tagging, quoted replies, and multi-attachment support.

## When to Use This

- When implementing chat functionality to send text messages between users
- When building file sharing features to send images, videos, audio, or documents
- When creating custom message types like location sharing, polls, or interactive cards
- When adding reply/quote functionality to reference previous messages
- When attaching metadata (like user location) to messages for additional context

## Prerequisites

- **SDK**: `@cometchat/chat-sdk-javascript` v4.x (v3.0.9+ for multi-attachment support)
- **State**: User must be logged in via `CometChat.login()` before sending messages
- **Receiver**: The recipient user (UID) or group (GUID) must exist in CometChat
- **For Media**: File object from input element or hosted URL for media messages

## Minimal Example

<Tabs>
<Tab title="JavaScript">
```javascript
const receiverID = "UID";
const messageText = "Hello world!";
const receiverType = CometChat.RECEIVER_TYPE.USER;

const textMessage = new CometChat.TextMessage(
  receiverID,
  messageText,
  receiverType
);

CometChat.sendMessage(textMessage).then(
  (message) => {
    console.log("Message sent successfully:", message);
  },
  (error) => {
    console.log("Message sending failed:", error);
  }
);
```
</Tab>

## Step-by-Step

Using CometChat, you can send three types of messages:

1. [Text Message](/sdk/javascript/send-message#text-message) is the most common and standard message type.
2. [Media Message](/sdk/javascript/send-message#media-message) for sending photos, videos and files.
3. [Custom Message](/sdk/javascript/send-message#custom-message), for sending completely custom data using JSON structures.
4. [Interactive Messages](/sdk/javascript/interactive-messages), for sending end-user interactive messages of type form, card and custom Interactive

You can also send metadata along with a text, media or custom message. Think, for example, if you'd want to share the user's location with every message, you can use the metadata field

## Text Message

*In other words, as a sender, how do I send a text message?*

To send a text message to a single user or group, you need to use the `sendMessage()` method and pass a `TextMessage` object to it.

### Add Metadata

To send custom data along with a text message, you can use the `setMetadata` method and pass a `JSON Object` to it.

<Tabs>
<Tab title="Add Metadata">
```javascript

  latitude: "50.6192171633316",
  longitude: "-72.68182268750002",
};

textMessage.setMetadata(metadata);
```

</Tab>

```typescript

```

</Tab>

</Tabs>

### Add Tags

To add a tag to a message you can use the `setTags()` method of the TextMessage Class. The `setTags()` method accepts a list of tags.

## Media Message

There are 2 ways you can send Media Messages using the CometChat SDK:

1. **By providing the File:** You can directly share the file object while creating an object of the MediaMessage class. When the media message is sent using the `sendMediaMessage()` method, this file is then uploaded to CometChat servers and the URL of the file is sent in the success response of the `sendMediaMessage()` function.

Getting file Object:

<Tabs>
<Tab title="HTML">
```html

      var UID = "UID";
      var file = document.getElementById("img_file").files[0];
      var fileType = MESSAGE_TYPE.IMAGE;
      var userType = CometChat.RECEIVER_TYPE.USER;
      var mediaMessage = new MediaMessage(UID, file, fileType, userType);
    </script>
  </body>
```

## Custom Message

*In other words, as a sender, how do I send a custom message like location coordinates?*

CometChat allows you to send custom messages which are neither text nor media messages.

In order to send a custom message, you need to use the `sendCustomMessage()` method.

The `sendCustomMessage()` method takes an object of the `CustomMessage` which can be obtained using the below constructor.

<Tabs>
<Tab title="JavaScript">
```js
let customMessage = new CometChat.CustomMessage(
  receiverID,
  receiverType,
  customType,
  customData
);
```

</Tab>

<Tab title="TypeScript">
```typescript

  receiverID,
  receiverType,
  customType,
  customData
);
```

</Tab>

</Tabs>

The above constructor, helps you create a custom message with the message type set to whatever is passed to the constructor and the category set to `custom`.

The parameters involved are:

1. `receiverId` - The unique ID of the user or group to which the message is to be sent.
2. `receiverType` - Type of the receiver i.e user or group
3. `customType` - custom message type that you need to set
4. `customData` - The data to be passed as the message in the form of a JSON Object.

You can also use the subType field of the `CustomMessage` class to set a specific type for the custom message. This can be achieved using the `setSubtype()` method.

### Update Conversation

```javascript
  receiverType,

);

customMessage.shouldUpdateConversation(false);
```

## API Details

### sendMessage (Text Messages)

#### Method Signature

`CometChat.sendMessage(textMessage: TextMessage): Promise<TextMessage>`

#### Parameters

| Parameter   | Type        | Required | Default | Description                                |
| ----------- | ----------- | -------- | ------- | ------------------------------------------ |
| textMessage | TextMessage | Yes      | -       | TextMessage object containing message data |

#### TextMessage Constructor Parameters

| Parameter    | Type   | Required | Description                                                              |
| ------------ | ------ | -------- | ------------------------------------------------------------------------ |
| receiverID   | string | Yes      | UID of the user or GUID of the group receiving the message               |
| messageText  | string | Yes      | The text content of the message                                          |
| receiverType | string | Yes      | `CometChat.RECEIVER_TYPE.USER` or `CometChat.RECEIVER_TYPE.GROUP`        |

#### Return Value

Returns a `Promise<CometChat.TextMessage>` that resolves to the sent TextMessage object containing message ID, sent timestamp, and delivery status.

### sendMediaMessage (Media Messages)

#### Method Signature

`CometChat.sendMediaMessage(mediaMessage: MediaMessage): Promise<MediaMessage>`

#### Parameters

| Parameter    | Type         | Required | Default | Description                                 |
| ------------ | ------------ | -------- | ------- | ------------------------------------------- |
| mediaMessage | MediaMessage | Yes      | -       | MediaMessage object containing file/URL data |

#### MediaMessage Constructor Parameters

| Parameter    | Type                | Required | Description                                                                                                                                     |
| ------------ | ------------------- | -------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| receiverID   | string              | Yes      | UID of the user or GUID of the group receiving the message                                                                                      |
| file         | File \| File[] \| string | Yes      | File object, array of files, or empty string (when using Attachment)                                                                            |
| messageType  | string              | Yes      | `CometChat.MESSAGE_TYPE.IMAGE`, `VIDEO`, `AUDIO`, or `FILE`                                                                                     |
| receiverType | string              | Yes      | `CometChat.RECEIVER_TYPE.USER` or `CometChat.RECEIVER_TYPE.GROUP`                                                                               |

#### Return Value

Returns a `Promise<CometChat.MediaMessage>` that resolves to the sent MediaMessage object containing the uploaded file URL.

### sendCustomMessage (Custom Messages)

#### Method Signature

`CometChat.sendCustomMessage(customMessage: CustomMessage): Promise<CustomMessage>`

#### Parameters

| Parameter     | Type          | Required | Default | Description                                  |
| ------------- | ------------- | -------- | ------- | -------------------------------------------- |
| customMessage | CustomMessage | Yes      | -       | CustomMessage object containing custom data  |

#### CustomMessage Constructor Parameters

| Parameter    | Type   | Required | Description                                                       |
| ------------ | ------ | -------- | ----------------------------------------------------------------- |
| receiverID   | string | Yes      | UID of the user or GUID of the group receiving the message        |
| receiverType | string | Yes      | `CometChat.RECEIVER_TYPE.USER` or `CometChat.RECEIVER_TYPE.GROUP` |
| customType   | string | Yes      | Custom message type identifier (e.g., "location", "poll")         |
| customData   | Object | Yes      | JSON object containing the custom message data                    |

#### Return Value

Returns a `Promise<CometChat.CustomMessage>` that resolves to the sent CustomMessage object.

### Common Errors

| Error Code                | Description                              | Resolution                                                    |
| ------------------------- | ---------------------------------------- | ------------------------------------------------------------- |
| ERR_NOT_LOGGED_IN         | User not logged in                       | Call `CometChat.login()` before sending messages              |
| ERR_UID_NOT_FOUND         | Receiver UID doesn't exist               | Verify the user exists in CometChat                           |
| ERR_GUID_NOT_FOUND        | Receiver GUID doesn't exist              | Verify the group exists in CometChat                          |
| ERR_NOT_A_MEMBER          | User is not a member of the group        | Join the group before sending messages                        |
| ERR_FILE_SIZE_EXCEEDED    | Media file exceeds size limit            | Compress the file or use a smaller file                       |
| ERR_INVALID_MESSAGE_TYPE  | Invalid message type specified           | Use valid MESSAGE_TYPE constants                              |

## Edge Cases & Pitfalls

<Warning>
**Pitfall 1: Not handling send failures gracefully**

Network issues or rate limits can cause message sends to fail. Always implement proper error handling and consider adding retry logic with exponential backoff. Store failed messages locally and provide users with a way to retry sending.
</Warning>

<Warning>
**Pitfall 2: Sending messages before login completes**

Attempting to send messages before `CometChat.login()` resolves will fail with `ERR_NOT_LOGGED_IN`. Always verify the user is logged in using `CometChat.getLoggedinUser()` before enabling send functionality.
</Warning>

<Warning>
**Pitfall 3: Using wrong receiver type**

Using `RECEIVER_TYPE.USER` with a GUID or `RECEIVER_TYPE.GROUP` with a UID will cause the message to fail. Always match the receiver type with the correct ID type (UID for users, GUID for groups).
</Warning>

<Warning>
**Pitfall 4: Large file uploads without progress indication**

Media message uploads can take time for large files. The `sendMediaMessage()` method doesn't provide upload progress. Consider showing a loading indicator and informing users that large files may take longer to send.
</Warning>

<Warning>
**Pitfall 5: Not validating file types before upload**

Sending unsupported file types may fail or cause issues. Validate file extensions and MIME types on the client side before calling `sendMediaMessage()`. Use the appropriate `MESSAGE_TYPE` constant for the file being sent.
</Warning>

## Performance & Scale Notes

- **Message rate limits**: CometChat enforces rate limits on message sending. Check [Rate Limits](/sdk/javascript/rate-limits) for current limits.
- **File size limits**: Media messages have file size limits based on your plan. Large files should be compressed or chunked before upload.
- **Multi-attachment support**: Available in SDK v3.0.9+. Sending multiple attachments in one message is more efficient than sending multiple separate messages.
- **Metadata size**: Keep metadata JSON objects small. Large metadata can impact message delivery performance.
- **Batch operations**: For sending messages to multiple recipients, consider using server-side APIs for better performance.

## Security Notes

<Note>
**Message Content Validation**

Always validate and sanitize user input before sending messages. Never include sensitive data (passwords, API keys, personal information) in message metadata or custom data fields.
</Note>

<Note>
**File Upload Security**

When using URL-based attachments, ensure the URLs point to trusted sources. Validate file types and sizes on both client and server side. Consider implementing virus scanning for uploaded files in production applications.
</Note>

<Warning>
**Custom Message Data**

Custom message data is visible to all recipients. Never include sensitive information in custom messages. Use server-side encryption for sensitive data that must be transmitted.
</Warning>
