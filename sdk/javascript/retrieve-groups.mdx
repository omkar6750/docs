## Summary

The CometChat JavaScript SDK enables you to retrieve groups in your application. You can fetch a list of all available groups, filter groups by various criteria, retrieve details of a specific group, or get the online member count for groups. This is essential for displaying group lists, implementing group search, and showing group information.

## When to Use This

- When displaying a list of available groups to users
- When implementing group search functionality
- When showing only groups the user has joined
- When filtering groups by tags or other criteria
- When retrieving detailed information about a specific group
- When showing online member counts for groups

## Prerequisites

- **SDK**: `@cometchat/chat-sdk-javascript` v4.x
- **State**: User must be logged in via `CometChat.login()`
- **Dashboard**: Ensure your CometChat app is created in the [CometChat Dashboard](https://app.cometchat.com)

## Minimal Example

<Tabs>
<Tab title="JavaScript">
```javascript
let limit = 30;
let groupsRequest = new CometChat.GroupsRequestBuilder()
  .setLimit(limit)
  .build();

  (groupList) => {
    console.log("Groups list fetched successfully", groupList);
  },

);
```

## Step-by-Step

### Step 1: Create a GroupsRequestBuilder

To fetch groups, you need to create a `GroupsRequest` object using the `GroupsRequestBuilder` class. The builder allows you to configure various parameters for filtering and pagination.

<Tabs>
<Tab title="JavaScript">
```javascript
let groupsRequestBuilder = new CometChat.GroupsRequestBuilder();
```
</Tab>
<Tab title="TypeScript">
```typescript
let groupsRequestBuilder: CometChat.GroupsRequestBuilder = new CometChat.GroupsRequestBuilder();
```
</Tab>
</Tabs>

### Step 2: Set the Limit

Use `setLimit()` to specify the number of groups to fetch in a single request. This is important for pagination.

<Tabs>
<Tab title="JavaScript">
```javascript
let limit = 30;
let groupsRequest = new CometChat.GroupsRequestBuilder()
  .setLimit(limit)
  .build();
```
</Tab>
<Tab title="TypeScript">
```typescript
let limit: number = 30;
let groupsRequest: CometChat.GroupsRequest = new CometChat.GroupsRequestBuilder()
  .setLimit(limit)
  .build();
```
</Tab>
</Tabs>

### Step 3: Add Search Filters (Optional)

### Step 4: Fetch Groups

Call `fetchNext()` on the `GroupsRequest` object to retrieve the groups. This method returns a Promise that resolves to an array of `Group` objects.

<Tabs>
<Tab title="JavaScript">
```javascript

let groupsRequest = new CometChat.GroupsRequestBuilder()
  .setLimit(limit)
  .build();

groupsRequest.fetchNext().then(
  (groupList) => {
    console.log("Groups list fetched successfully", groupList);
  },
  (error) => {
    console.log("Groups list fetching failed with error", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript

let groupsRequest: CometChat.GroupsRequest = new CometChat.GroupsRequestBuilder()
  .setLimit(limit)
  .build();

  (groupList: CometChat.Group[]) => {
    console.log("Groups list fetched successfully", groupList);

  (error: CometChat.CometChatException) => {
```

</Tab>

### Step 5: Implement Pagination

To fetch the next set of groups, call `fetchNext()` again on the same `GroupsRequest` object. The SDK maintains the pagination state internally.

<Tabs>
<Tab title="JavaScript">
```javascript
let limit = 30;
let groupsRequest = new CometChat.GroupsRequestBuilder()
  .setLimit(limit)
  .build();

// First page
groupsRequest.fetchNext().then(
  (groupList) => {
    console.log("First page of groups:", groupList);
    
    // Fetch next page when needed
    return groupsRequest.fetchNext();
  }
).then(
  (nextGroupList) => {
    console.log("Second page of groups:", nextGroupList);
  },
  (error) => {
    console.log("Error fetching groups:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript

  .setLimit(limit)

    console.log("Second page of groups:", nextGroupList);
```

```javascript
    console.log("Group details fetching failed with exception:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript

CometChat.getGroup(GUID).then(
  (group: CometChat.Group) => {
    console.log("Group details fetched successfully:", group);
  },
  (error: CometChat.CometChatException) => {

);
```
</Tab>

## API Details

### Method Signatures

```javascript
// Fetch groups list
new CometChat.GroupsRequestBuilder().setLimit(limit).build(): GroupsRequest
groupsRequest.fetchNext(): Promise<Group[]>

// Get specific group
CometChat.getGroup(GUID: string): Promise<Group>

// Get online member count
CometChat.getOnlineGroupMemberCount(guids: string[]): Promise<Object>
```

### GroupsRequestBuilder Methods

| Method             | Type                  | Description                                                    |
| ------------------ | --------------------- | -------------------------------------------------------------- |
| setLimit           | number                | Sets the number of groups to fetch per request (max 100)       |
| setSearchKeyword   | string                | Filters groups by name matching the search keyword             |
| joinedOnly         | boolean               | When true, returns only groups the user has joined             |
| setTags            | string[]              | Filters groups that have any of the specified tags             |
| withTags           | boolean               | When true, includes tags data in the returned Group objects    |
| build              | -                     | Builds and returns the GroupsRequest object                    |

### Parameters

#### getGroup

| Parameter | Type   | Required | Description                                      |
| --------- | ------ | -------- | ------------------------------------------------ |
| GUID      | string | Yes      | The unique identifier of the group to retrieve   |

#### getOnlineGroupMemberCount

| Parameter | Type     | Required | Description                                      |
| --------- | -------- | -------- | ------------------------------------------------ |
| guids     | string[] | Yes      | Array of GUIDs to get online member counts for   |

### Return Values

- `fetchNext()`: Returns a `Promise<Group[]>` that resolves to an array of Group objects
- `getGroup()`: Returns a `Promise<Group>` that resolves to a single Group object
- `getOnlineGroupMemberCount()`: Returns a `Promise<Object>` with GUIDs as keys and online counts as values

### Group Object Properties

| Property     | Type    | Description                                                    |
| ------------ | ------- | -------------------------------------------------------------- |
| guid         | string  | Unique identifier for the group                                |
| name         | string  | Name of the group                                              |
| type         | string  | Group type: public, password, or private                       |
| icon         | string  | URL to the group's icon                                        |
| description  | string  | Description of the group                                       |
| owner        | string  | UID of the group owner                                         |
| membersCount | number  | Total number of members in the group                           |
| hasJoined    | boolean | Whether the logged-in user is a member of the group            |
| joinedAt     | number  | Unix timestamp when the user joined (if member)                |
| scope        | string  | User's scope in the group (admin, moderator, participant)      |
| tags         | string[]| Tags associated with the group (when withTags is true)         |
| createdAt    | number  | Unix timestamp when the group was created                      |
| updatedAt    | number  | Unix timestamp when the group was last updated                 |

### Errors

| Error Code              | Description                              | Resolution                                    |
| ----------------------- | ---------------------------------------- | --------------------------------------------- |
| ERR_USER_NOT_LOGGED_IN  | No user is currently logged in           | Ensure user is logged in before fetching      |
| ERR_INVALID_GUID        | The provided GUID is invalid             | Verify the GUID format and existence          |
| ERR_GROUP_NOT_FOUND     | Group with the specified GUID not found  | Check if the group exists                     |

## Edge Cases & Pitfalls

<Warning>
**Private Groups Visibility**: Private groups are only returned in the groups list if the logged-in user is a member. Non-members cannot see or search for private groups. If you expect to see a private group but don't, verify the user's membership status.
</Warning>

<Warning>
**Pagination State**: The `GroupsRequest` object maintains internal pagination state. If you need to restart from the beginning, create a new `GroupsRequestBuilder` and build a fresh `GroupsRequest` object. Reusing the same object will continue from where it left off.
</Warning>

<Warning>
**Empty Results Don't Mean End**: When `fetchNext()` returns an empty array, it means there are no more groups to fetch. However, if you're using filters like `joinedOnly(true)` or `setTags()`, an empty result might also mean no groups match your criteria. Always handle both scenarios appropriately.
</Warning>

<Warning>
**Search Keyword Case Sensitivity**: The `setSearchKeyword()` method performs a case-insensitive search on group names. However, it only matches from the beginning of words, not substrings within words.
</Warning>

## Performance & Scale Notes

- The maximum limit per request is 100 groups. Setting a higher value will be capped at 100.
- Use pagination to handle large numbers of groups efficiently rather than fetching all at once.
- The `joinedOnly(true)` filter is more efficient than fetching all groups and filtering client-side.
- `getOnlineGroupMemberCount()` accepts multiple GUIDs to batch requests and reduce API calls.
- Consider caching group lists locally and refreshing periodically rather than fetching on every view.

## Security Notes

<Note>
Group retrieval operations use the logged-in user's authentication. No additional Auth Key is required for these operations.
</Note>

<Warning>
**Password-Protected Groups**: Password-protected groups appear in search results, but users must provide the correct password to join. The password itself is never exposed through the API.
</Warning>

<Warning>
**Group Metadata**: If your groups contain sensitive information in metadata, be aware that metadata is returned with group objects. Design your metadata schema with this visibility in mind.
</Warning>
