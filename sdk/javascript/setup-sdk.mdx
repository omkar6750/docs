## Summary

This page guides you through setting up the CometChat JavaScript SDK in your application. You'll learn how to obtain your application credentials, install the SDK via NPM or CDN, and initialize CometChat to enable real-time messaging and calling features.

## When to Use This

- When starting a new project that requires real-time chat functionality
- When integrating CometChat into an existing JavaScript or TypeScript application
- When migrating from CometChat SDK v3 to v4
- When setting up CometChat in a web application (React, Vue, Angular, or vanilla JS)
- When configuring CometChat for a Node.js server-side application

## Prerequisites

- A CometChat account ([Sign up here](https://app.cometchat.com))
- A CometChat app with App ID, Region, and Auth Key from the Dashboard
- Node.js and npm (for NPM installation) or a web browser (for CDN installation)
- Basic knowledge of JavaScript or TypeScript

<Note>
Migrating app version from v3 to v4?

Skip the create new app step. Your existing v3 app can be migrated to v4.

Follow steps mentioned in **Add the CometChat dependency** section below to upgrade to latest version of v4.
</Note>

[Signup for CometChat](https://app.cometchat.com) and then:

## Initialize CometChat

The `init()` method initializes the settings required for CometChat. The `init()` method takes the below parameters:

1. appID - Your CometChat App ID
2. appSettings - An object of the AppSettings class can be created using the AppSettingsBuilder class. The region field is mandatory and can be set using the `setRegion()` method.

The `AppSettings` class allows you to configure the following settings:

- **Region**: The region where your app was created.
- **[Presence Subscription](/sdk/javascript/user-presence)**: Represents the subscription type for user presence (real-time online/offline status)
- **autoEstablishSocketConnection(boolean value)**: This property takes a boolean value which when set to `true` informs the SDK to manage the web-socket connection internally. If set to `false`, it informs the SDK that the web-socket connection will be managed manually. The default value for this parameter is `true`. For more information on this, please check the [Managing Web-Socket connections manually](/sdk/javascript/managing-web-sockets-connections-manually) section.
- **overrideAdminHost(adminHost: string)**: This method takes the admin URL as input and uses this admin URL instead of the default admin URL. This can be used in case of dedicated deployment of CometChat.
- **overrideClientHost(clientHost: string)**: This method takes the client URL as input and uses this client URL instead of the default client URL. This can be used in case of dedicated deployment of CometChat.

You need to call `init()` before calling any other method from CometChat. We suggest you call the `init()` method on app startup, preferably in the `index.js` file.

<Tabs>
<Tab title="JavaScript">
```javascript
const APP_ID = "APP_ID";
const REGION = "REGION";

const appSetting = new CometChat.AppSettingsBuilder()
  .subscribePresenceForAllUsers()
  .setRegion(REGION)
  .autoEstablishSocketConnection(true)
  .build();

CometChat.init(APP_ID, appSetting).then(
  () => {
    console.log("Initialization completed successfully");
  },
  (error) => {
    console.log("Initialization failed with error:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript
const APP_ID: string = "APP_ID";
const REGION: string = "REGION";

const appSetting: CometChat.AppSettings = new CometChat.AppSettingsBuilder()
  .subscribePresenceForAllUsers()
  .setRegion(REGION)
  .autoEstablishSocketConnection(true)
  .build();

CometChat.init(APP_ID, appSetting).then(
  (initialized: boolean) => {
    console.log("Initialization completed successfully", initialized);
  },
  (error: CometChat.CometChatException) => {
    console.log("Initialization failed with error:", error);
  }
);
```
</Tab>
</Tabs>

Make sure you replace `APP_ID` with your CometChat **App ID** and `REGION` with your **App Region** in the above code.

### Parameters

| Parameter   | Type        | Required | Description                         |
| ----------- | ----------- | -------- | ----------------------------------- |
| appID       | string      | Yes      | Your CometChat App ID               |
| appSetting  | AppSettings | Yes      | An object of the AppSettings class  |

### Return Value

Returns a `Promise<boolean>` that resolves to `true` when initialization is successful.

### Errors

| Error Code | Description | Resolution |
| ---------- | ----------- | ---------- |
| ERR_INVALID_APP_ID | Invalid or missing App ID | Verify your App ID from the CometChat Dashboard |
| ERR_INVALID_REGION | Invalid or missing region | Ensure region matches your app's region (us, eu, in) |
| ERR_NETWORK_ERROR | Network connectivity issue | Check internet connection and try again |

## Edge Cases & Pitfalls

<Warning>
**SSR/Server-Side Rendering Issues**: CometChat SDK requires browser APIs (like `window` and `WebSocket`) that are not available in server-side environments. When using frameworks like Next.js, Nuxt, or SvelteKit, ensure you only initialize CometChat on the client side. Use dynamic imports or check for `typeof window !== 'undefined'` before calling `init()`.
</Warning>

<Warning>
**Init Timing**: Always wait for the `init()` promise to resolve before calling any other CometChat methods like `login()` or `getLoggedinUser()`. Calling methods before initialization completes will result in errors. Consider using async/await or promise chaining to ensure proper sequencing.
</Warning>

<Warning>
**Multiple Init Calls**: Calling `init()` multiple times (e.g., on component re-renders) can cause unexpected behavior. Initialize CometChat only once at the application entry point. Use a flag or singleton pattern to prevent duplicate initialization.
</Warning>

<Warning>
**Region Mismatch**: Ensure the region you specify in `setRegion()` matches the region where your CometChat app was created. Using the wrong region will cause connection failures. Check your app's region in the CometChat Dashboard under App Settings.
</Warning>

## Security Notes

<Note>
**Auth Key** is acceptable for POC/development only. For production applications, use **Auth Token** created server-side via the [CometChat REST API](/reference/createauthtoken). Never expose your Auth Key in production client-side code as it can be used to impersonate any user.
</Note>

<Note>
**REST API Key** should never be included in client-side code. Keep it secure on your server and use it only for server-to-server API calls.
</Note>
