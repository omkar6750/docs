This guide covers the complete setup process for the CometChat JavaScript SDK, including installation, initialization, and framework-specific configurations.

<Info>
**Migrating from v3?** Your existing app can be upgraded to v4. See the [Migration Guide](/sdk/javascript/upgrading-from-v3).
</Info>

## Prerequisites

Before you begin, ensure you have:

<Steps>
  <Step title="CometChat Account">
    [Sign up](https://app.cometchat.com) or log in to your CometChat dashboard
  </Step>
  <Step title="App Credentials">
    From your app's **API & Auth Keys** section, note:
    - **App ID** - Your unique application identifier
    - **Region** - Where your app is hosted (`us`, `eu`, or `in`)
    - **Auth Key** - For development/testing (use Auth Tokens in production)
  </Step>
</Steps>

## Installation

Choose your preferred installation method:

<Tabs>
  <Tab title="npm">
    ```bash
    npm install @cometchat/chat-sdk-javascript
    ```
    
    Then import in your code:
    ```javascript
    import { CometChat } from "@cometchat/chat-sdk-javascript";
    ```
  </Tab>
  <Tab title="yarn">
    ```bash
    yarn add @cometchat/chat-sdk-javascript
    ```
    
    Then import in your code:
    ```javascript
    import { CometChat } from "@cometchat/chat-sdk-javascript";
    ```
  </Tab>
  <Tab title="CDN">
    Add to your HTML:
    ```html
    <script 
      type="text/javascript" 
      src="https://unpkg.com/@cometchat/chat-sdk-javascript/CometChat.js">
    </script>
    ```
    
    The `CometChat` object is available globally.
  </Tab>
</Tabs>

## Initialize CometChat

Call `init()` once when your application starts, before using any other CometChat methods.

<Tabs>
  <Tab title="JavaScript">
    ```javascript
    const appID = "YOUR_APP_ID";
    const region = "YOUR_REGION";

    const appSetting = new CometChat.AppSettingsBuilder()
      .subscribePresenceForAllUsers()
      .setRegion(region)
      .autoEstablishSocketConnection(true)
      .build();

    CometChat.init(appID, appSetting).then(
      () => {
        console.log("Initialization completed successfully");
      },
      (error) => {
        console.log("Initialization failed with error:", error);
      }
    );
    ```
  </Tab>
  <Tab title="TypeScript">
    ```typescript
    const appID: string = "YOUR_APP_ID";
    const region: string = "YOUR_REGION";

    const appSetting: CometChat.AppSettings = new CometChat.AppSettingsBuilder()
      .subscribePresenceForAllUsers()
      .setRegion(region)
      .autoEstablishSocketConnection(true)
      .build();

    CometChat.init(appID, appSetting).then(
      (initialized: boolean) => {
        console.log("Initialization completed successfully", initialized);
      },
      (error: CometChat.CometChatException) => {
        console.log("Initialization failed with error:", error);
      }
    );
    ```
  </Tab>
</Tabs>

<Warning>
Replace `YOUR_APP_ID` and `YOUR_REGION` with your actual credentials from the CometChat dashboard.
</Warning>

## Configuration Options

The `AppSettingsBuilder` provides several configuration options:

| Method | Description | Default |
|--------|-------------|---------|
| `setRegion(region)` | **Required.** Set your app's region (`us`, `eu`, `in`) | - |
| `subscribePresenceForAllUsers()` | Receive presence updates for all users | Disabled |
| `subscribePresenceForRoles(roles)` | Receive presence for specific roles only | - |
| `subscribePresenceForFriends()` | Receive presence for friends only | - |
| `autoEstablishSocketConnection(bool)` | SDK manages WebSocket connection | `true` |
| `setStorageMode(mode)` | Set data storage mode | Local Storage |
| `overrideAdminHost(url)` | Custom admin URL (dedicated deployment) | - |
| `overrideClientHost(url)` | Custom client URL (dedicated deployment) | - |

### Storage Modes

Control how CometChat stores data locally:

```javascript
const appSetting = new CometChat.AppSettingsBuilder()
  .setRegion(region)
  .setStorageMode(CometChat.StorageMode.SESSION) // Session storage
  .build();
```

Available modes:
- `CometChat.StorageMode.LOCAL` - Persists across browser sessions (default)
- `CometChat.StorageMode.SESSION` - Cleared when browser session ends

### Presence Subscription

Choose how to receive user online/offline status:

<Tabs>
  <Tab title="All Users">
    ```javascript
    // Receive presence for all users
    const appSetting = new CometChat.AppSettingsBuilder()
      .setRegion(region)
      .subscribePresenceForAllUsers()
      .build();
    ```
  </Tab>
  <Tab title="Specific Roles">
    ```javascript
    // Receive presence for specific roles only
    const appSetting = new CometChat.AppSettingsBuilder()
      .setRegion(region)
      .subscribePresenceForRoles(["premium", "moderator"])
      .build();
    ```
  </Tab>
  <Tab title="Friends Only">
    ```javascript
    // Receive presence for friends only
    const appSetting = new CometChat.AppSettingsBuilder()
      .setRegion(region)
      .subscribePresenceForFriends()
      .build();
    ```
  </Tab>
</Tabs>

Learn more about [User Presence](/sdk/javascript/user-presence).

## Framework-Specific Setup

### Next.js (SSR)

For Next.js applications, import CometChat dynamically on the client side.

**App Router (React Server Components):**

```javascript
// app/chat/page.js
"use client";

import { useEffect, useState } from "react";

export default function ChatPage() {
  const [isReady, setIsReady] = useState(false);

  useEffect(() => {
    const initCometChat = async () => {
      const { CometChat } = await import("@cometchat/chat-sdk-javascript");
      
      const appSetting = new CometChat.AppSettingsBuilder()
        .subscribePresenceForAllUsers()
        .setRegion("YOUR_REGION")
        .build();

      await CometChat.init("YOUR_APP_ID", appSetting);
      setIsReady(true);
    };

    initCometChat();
  }, []);

  if (!isReady) return <div>Loading...</div>;

  return <div>Chat is ready!</div>;
}
```

**Pages Router:**

```javascript
// pages/index.js
import { useEffect, useState } from "react";
import Chat from "../components/Chat";

export default function Home() {
  const [libraryImported, setLibraryImported] = useState(false);

  useEffect(() => {
    window.CometChat = require("@cometchat/chat-sdk-javascript").CometChat;
    setLibraryImported(true);
  }, []);

  return libraryImported ? <Chat /> : <p>Loading...</p>;
}
```

### Nuxt.js (SSR)

For Nuxt applications, import CometChat in the `mounted` lifecycle hook:

```vue
<template>
  <div v-if="libraryImported">
    <Chat />
  </div>
  <div v-else>Loading...</div>
</template>

<script>
export default {
  name: "Index",
  components: {
    Chat: () => import("./Chat.vue"),
  },
  data() {
    return {
      libraryImported: false,
    };
  },
  mounted() {
    window.CometChat = require("@cometchat/chat-sdk-javascript").CometChat;
    this.libraryImported = true;
  },
};
</script>
```

## Verify Installation

After setup, verify CometChat is working:

```javascript
CometChat.init(appID, appSetting).then(
  () => {
    console.log("CometChat initialized");
    
    // Try logging in with a test user
    return CometChat.login("cometchat-uid-1", authKey);
  }
).then(
  (user) => {
    console.log("Login successful:", user.getName());
  },
  (error) => {
    console.error("Error:", error);
  }
);
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Initialization fails with 'App not found'">
    Verify your App ID and Region match exactly what's shown in your CometChat dashboard. Region values are case-sensitive (`us`, `eu`, `in`).
  </Accordion>
  <Accordion title="'CometChat is not defined' error">
    Ensure you've imported CometChat before using it. For CDN usage, make sure the script tag loads before your code runs.
  </Accordion>
  <Accordion title="SSR/Hydration errors in Next.js or Nuxt">
    CometChat requires browser APIs. Use dynamic imports or load the SDK only on the client side. See the framework-specific sections above.
  </Accordion>
</AccordionGroup>
