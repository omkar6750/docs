title: "Delivery & Read Receipts"
---

## Summary

The CometChat JavaScript SDK enables you to track message delivery and read status in real-time. You can mark messages as delivered when they reach a recipient's device, mark them as read when viewed, and receive real-time notifications when your sent messages are delivered or read by others.

## When to Use This

- When you need to show delivery checkmarks (✓✓) indicating a message reached the recipient's device
- When you need to show read indicators (blue checkmarks) when a message has been viewed
- When building a chat UI that displays message status (sent, delivered, read)
- When you want to mark entire conversations as delivered or read in bulk
- When implementing unread message counts and "mark as unread" functionality

## Prerequisites

- **SDK**: `@cometchat/chat-sdk-javascript` v4.x
- **State**: User must be logged in via `CometChat.login()` before using receipt methods
- **Messages**: Messages must exist in the conversation to mark them as delivered/read
- **Enhanced Messaging Status**: Enable in Dashboard for `onMessagesDeliveredToAll`, `onMessagesReadByAll`, and `markMessageAsUnread` features

## Minimal Example

<Tabs>
<Tab title="JavaScript">
```javascript
// Mark a message as read
const messageId = "MESSAGE_ID";
const receiverId = "UID";
const receiverType = "user";
const senderId = "UID";

CometChat.markAsRead(messageId, receiverId, receiverType, senderId).then(
  () => {
    console.log("Message marked as read");
  },
  (error) => {
    console.log("Error marking message as read:", error);
  }
);
```
</Tab>

## Step-by-Step

### Mark Messages as Delivered

As a recipient, you can inform the sender that you've received a message using the `markAsDelivered()` method. This method accepts the following parameters:

| Parameter      | Type   | Description                                                                                                                                                                         |
| -------------- | ------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `messageId`    | string | The ID of the message above which all messages for the conversation are marked as delivered.                                                                                        |
| `receiverId`   | string | For one-to-one conversations, use the sender's `UID`. For group conversations, use the group's `GUID`.                                                                              |
| `receiverType` | string | Type of the receiver: `user` or `group`.                                                                                                                                            |
| `senderId`     | string | The `UID` of the message sender.                                                                                                                                                    |

Ideally, mark messages as delivered in two scenarios:

1. **When fetching the message list**: Pass the last message's ID to `markAsDelivered()`.
2. **When receiving a real-time message**: Pass the incoming message's ID to `markAsDelivered()`.

<Tabs>
<Tab title="JavaScript">
```javascript
// Mark as delivered for user conversation
const messageId = "MESSAGE_ID";
const receiverId = "UID";
const receiverType = "user";
const senderId = "UID";

CometChat.markAsDelivered(messageId, receiverId, receiverType, senderId).then(
  () => {
    console.log("Mark as delivered success");
  },
  (error) => {
    console.log("Error marking as delivered:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript
// Mark as delivered for user conversation
const messageId: string = "MESSAGE_ID";
const receiverId: string = "UID";
const receiverType: string = "user";
const senderId: string = "UID";

  () => {
    console.log("Mark as delivered success");
  },
  (error: CometChat.CometChatException) => {

  }
);
```
</Tab>
</Tabs>

</Tab>

### Mark Messages as Unread

The `markMessageAsUnread()` method allows users to designate specific messages as unread, even if previously viewed. All messages below the specified message will contribute to the unread count.

<Note>
You cannot mark your own messages as unread. This method only works for messages received from other users.
</Note>

<Tabs>
<Tab title="JavaScript">
```javascript
CometChat.markMessageAsUnread(message).then(
  (conversation) => {
    console.log("Mark as unread success:", conversation);
    console.log("Unread message count:", conversation.getUnreadMessageCount());
  },
  (error) => {
    console.log("Error marking as unread:", error);
  }
);
```
</Tab>
<Tab title="TypeScript">
```typescript
const message: CometChat.BaseMessage = receivedMessage;

CometChat.markMessageAsUnread(message).then(
  (conversation: CometChat.Conversation) => {
    console.log("Mark as unread success:", conversation);
    console.log("Unread message count:", conversation.getUnreadMessageCount());
  },
  (error: CometChat.CometChatException) => {
    console.log("Error marking as unread:", error);
  }
);
```
</Tab>
</Tabs>

## Events & Real-time Behavior

### Registering Receipt Listeners

Listen for real-time delivery and read receipt events using `addMessageListener()`:

<Tabs>
<Tab title="JavaScript">
```javascript
const listenerId = "RECEIPT_LISTENER_ID";

CometChat.addMessageListener(
  listenerId,
  new CometChat.MessageListener({
    onMessagesDelivered: (messageReceipt) => {
      console.log("Message delivered:", messageReceipt);
    },
    onMessagesRead: (messageReceipt) => {
      console.log("Message read:", messageReceipt);
    },
    onMessagesDeliveredToAll: (messageReceipt) => {
      // Group conversations only
      console.log("Message delivered to all group members:", messageReceipt);
    },
    onMessagesReadByAll: (messageReceipt) => {
      // Group conversations only
      console.log("Message read by all group members:", messageReceipt);
    },
  })
);
```
</Tab>
<Tab title="TypeScript">
```typescript
const listenerId: string = "RECEIPT_LISTENER_ID";

CometChat.addMessageListener(
  listenerId,
  new CometChat.MessageListener({
    onMessagesDelivered: (messageReceipt: CometChat.MessageReceipt) => {
      console.log("Message delivered:", messageReceipt);
    },
    onMessagesRead: (messageReceipt: CometChat.MessageReceipt) => {
      console.log("Message read:", messageReceipt);
    },
    onMessagesDeliveredToAll: (messageReceipt: CometChat.MessageReceipt) => {
      // Group conversations only
      console.log("Message delivered to all group members:", messageReceipt);
    },
    onMessagesReadByAll: (messageReceipt: CometChat.MessageReceipt) => {
      // Group conversations only
      console.log("Message read by all group members:", messageReceipt);
    },
  })
);
```
</Tab>
</Tabs>

### MessageReceipt Object

Receipt events return a `MessageReceipt` object with the following properties:

| Property       | Type   | Description                                                                                                                               |
| -------------- | ------ | ----------------------------------------------------------------------------------------------------------------------------------------- |
| `messageId`    | number | The ID of the message prior to which all messages have been marked as delivered/read.                                                     |
| `sender`       | User   | User object of who marked the message. System User for `deliveredToAll` & `readByAll` events.                                             |
| `receiverId`   | string | ID of the receiver whose conversation was marked.                                                                                         |
| `receiverType` | string | Type of the receiver (`user` or `group`).                                                                                                 |
| `receiptType`  | string | Type of the receipt (`read` or `delivered`).                                                                                              |
| `deliveredAt`  | number | Timestamp when the message was delivered. Present only for `delivered` receipt type.                                                      |
| `readAt`       | number | Timestamp when the message was read. Present only for `read` receipt type.                                                                |

### Missed Receipts

When fetching messages in bulk (offline messages), the message object includes `deliveredAt` and `readAt` fields containing timestamps for when the message was delivered and read. Use these fields to determine delivery and read status for historical messages.

## API Details

### Method Signatures

```typescript
// Mark individual message as delivered
CometChat.markAsDelivered(
  messageId: string,
  receiverId: string,
  receiverType: string,
  senderId: string
): Promise<void>

// Mark message object as delivered
CometChat.markAsDelivered(message: BaseMessage): Promise<void>

// Mark conversation as delivered
CometChat.markConversationAsDelivered(
  conversationWith: string,
  conversationType: string
): Promise<string>

// Mark individual message as read
CometChat.markAsRead(
  messageId: string,
  receiverId: string,
  receiverType: string,
  senderId: string
): Promise<void>

// Mark message object as read
CometChat.markAsRead(message: BaseMessage): Promise<void>

// Mark conversation as read
CometChat.markConversationAsRead(
  conversationWith: string,
  conversationType: string
): Promise<string>

// Mark message as unread
CometChat.markMessageAsUnread(message: BaseMessage): Promise<Conversation>

// Get message receipts
CometChat.getMessageReceipts(messageId: number): Promise<MessageReceipt[]>
```

### Parameters

| Method                         | Parameter          | Type        | Required | Description                                      |
| ------------------------------ | ------------------ | ----------- | -------- | ------------------------------------------------ |
| `markAsDelivered`              | `messageId`        | string      | Yes      | ID of the message to mark as delivered           |
| `markAsDelivered`              | `receiverId`       | string      | Yes      | UID (user) or GUID (group) of the conversation   |
| `markAsDelivered`              | `receiverType`     | string      | Yes      | `user` or `group`                                |
| `markAsDelivered`              | `senderId`         | string      | Yes      | UID of the message sender                        |
| `markConversationAsDelivered`  | `conversationWith` | string      | Yes      | UID or GUID of the conversation                  |
| `markConversationAsDelivered`  | `conversationType` | string      | Yes      | `user` or `group`                                |
| `markAsRead`                   | `messageId`        | string      | Yes      | ID of the message to mark as read                |
| `markAsRead`                   | `receiverId`       | string      | Yes      | UID (user) or GUID (group) of the conversation   |
| `markAsRead`                   | `receiverType`     | string      | Yes      | `user` or `group`                                |
| `markAsRead`                   | `senderId`         | string      | Yes      | UID of the message sender                        |
| `markConversationAsRead`       | `conversationWith` | string      | Yes      | UID or GUID of the conversation                  |
| `markConversationAsRead`       | `conversationType` | string      | Yes      | `user` or `group`                                |
| `markMessageAsUnread`          | `message`          | BaseMessage | Yes      | Message object to mark as unread                 |
| `getMessageReceipts`           | `messageId`        | number      | Yes      | ID of the message to get receipts for            |

### Return Values

| Method                        | Returns                    | Description                                                |
| ----------------------------- | -------------------------- | ---------------------------------------------------------- |
| `markAsDelivered`             | `Promise<void>`            | Resolves when message is marked as delivered               |
| `markConversationAsDelivered` | `Promise<string>`          | Resolves with success message                              |
| `markAsRead`                  | `Promise<void>`            | Resolves when message is marked as read                    |
| `markConversationAsRead`      | `Promise<string>`          | Resolves with success message                              |
| `markMessageAsUnread`         | `Promise<Conversation>`    | Resolves with updated Conversation object                  |
| `getMessageReceipts`          | `Promise<MessageReceipt[]>`| Resolves with array of MessageReceipt objects              |

### Errors

| Error Code                  | Description                                    | Resolution                                      |
| --------------------------- | ---------------------------------------------- | ----------------------------------------------- |
| `ERR_NOT_LOGGED_IN`         | User is not logged in                          | Call `CometChat.login()` before using receipts  |
| `ERR_INVALID_MESSAGE_ID`    | Invalid or non-existent message ID             | Verify the message ID exists                    |
| `ERR_INVALID_RECEIVER_TYPE` | Invalid receiver type provided                 | Use `user` or `group` as receiver type          |
| `ERR_FEATURE_NOT_ENABLED`   | Enhanced Messaging Status not enabled          | Enable feature in CometChat Dashboard           |

## Edge Cases & Pitfalls

<Warning>
**Duplicate Listener Registration**: Registering multiple listeners with the same `listenerId` will overwrite the previous listener. Always use unique listener IDs for different components or features.
</Warning>

<Warning>
**Memory Leaks from Unremoved Listeners**: Failing to remove listeners when a component unmounts or when navigating away from a chat screen can cause memory leaks and duplicate event handling. Always call `removeMessageListener()` in cleanup logic.
</Warning>

<Warning>
**Cannot Mark Own Messages as Unread**: The `markMessageAsUnread()` method only works for messages received from other users. Attempting to mark your own sent messages as unread will fail silently.
</Warning>

<Warning>
**Enhanced Messaging Status Required**: The `onMessagesDeliveredToAll`, `onMessagesReadByAll` events, and `markMessageAsUnread()` method require the **Enhanced Messaging Status** feature to be enabled in your CometChat Dashboard.
</Warning>

<Warning>
**Group vs User Receiver ID**: For user conversations, use the sender's UID as `receiverId`. For group conversations, use the group's GUID. Using the wrong ID will result in receipts not being recorded correctly.
</Warning>
