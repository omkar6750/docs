## Summary

The CometChat JavaScript SDK enables you to manage group membership by kicking, banning, and unbanning members. Admins and moderators can remove disruptive users temporarily (kick) or permanently (ban), and restore access to previously banned users (unban). This is essential for building moderation tools, enforcing community guidelines, and maintaining healthy group environments.

## When to Use This

- When implementing moderation tools for group admins and moderators
- When building a "Remove Member" feature in group settings
- When enforcing community guidelines by banning rule violators
- When restoring access to previously banned users after appeals
- When creating admin dashboards for group management
- When implementing automated moderation based on user behavior

## Prerequisites

- **SDK**: `@cometchat/chat-sdk-javascript` v4.x
- **State**: User must be logged in via `CometChat.login()`
- **Dashboard**: Ensure your CometChat app is created in the [CometChat Dashboard](https://app.cometchat.com)
- **Group**: The group must exist and the logged-in user must be an admin or moderator
- **Permissions**: Only admins and moderators can kick, ban, or unban members

## Step-by-Step

### Step 1: Kick a Group Member

The Admin or Moderator of a group can kick a member out of the group using the `kickGroupMember()` method. A kicked user is removed from the group but can rejoin later.

<Tabs>
<Tab title="JavaScript">
```javascript

let UID = "UID";

  }
```

### Step 4: Fetch Banned Members List

To retrieve the list of banned members for a group, use the `BannedMembersRequestBuilder` class.

<Tabs>
<Tab title="JavaScript">
```javascript

  .setLimit(limit)
  .build();

  (bannedMembers) => {
    console.log("Banned members list fetched successfully:", bannedMembers);
  },
  (error) => {
    console.log("Banned members list fetching failed with error:", error);
  }
);
```

## Events & Real-time Behavior

### Registering GroupListener

To receive real-time events when members are kicked, banned, or unbanned, register a `GroupListener`.

<Tabs>
<Tab title="JavaScript">
```javascript
let listenerId = "unique-kick-ban-listener";

CometChat.addGroupListener(
  listenerId,
  new CometChat.GroupListener({
    onGroupMemberKicked: (message, kickedUser, kickedBy, kickedFrom) => {
      console.log("User kicked:", { message, kickedUser, kickedBy, kickedFrom });
    },
    onGroupMemberBanned: (message, bannedUser, bannedBy, bannedFrom) => {
      console.log("User banned:", { message, bannedUser, bannedBy, bannedFrom });
    },
    onGroupMemberUnbanned: (message, unbannedUser, unbannedBy, unbannedFrom) => {
      console.log("User unbanned:", { message, unbannedUser, unbannedBy, unbannedFrom });
    },
  })
);
```
</Tab>
<Tab title="TypeScript">
```typescript
let listenerId: string = "unique-kick-ban-listener";

CometChat.addGroupListener(
  listenerId,
  new CometChat.GroupListener({
    onGroupMemberKicked: (
      message: CometChat.Action,
      kickedUser: CometChat.User,
      kickedBy: CometChat.User,
      kickedFrom: CometChat.Group
    ) => {
      console.log("User kicked:", { message, kickedUser, kickedBy, kickedFrom });
    },
    onGroupMemberBanned: (
      message: CometChat.Action,
      bannedUser: CometChat.User,
      bannedBy: CometChat.User,
      bannedFrom: CometChat.Group
    ) => {
      console.log("User banned:", { message, bannedUser, bannedBy, bannedFrom });
    },
    onGroupMemberUnbanned: (
      message: CometChat.Action,
      unbannedUser: CometChat.User,
      unbannedBy: CometChat.User,
      unbannedFrom: CometChat.Group
    ) => {
      console.log("User unbanned:", { message, unbannedUser, unbannedBy, unbannedFrom });
    },
  })
);
```
</Tab>

### Kick/Ban/Unban Events in Message History

When retrieving previous messages, if a member was kicked, banned, or unbanned, the message list will contain an `Action` message. The `Action` object provides the following information:

| Action Type | action Value | Description                                      |
| ----------- | ------------ | ------------------------------------------------ |
| Kicked      | `"kicked"`   | A member was kicked from the group               |
| Banned      | `"banned"`   | A member was banned from the group               |
| Unbanned    | `"unbanned"` | A previously banned member was unbanned          |

**Action Object Properties:**

| Property  | Description                                                |
| --------- | ---------------------------------------------------------- |
| action    | The action type: `"kicked"`, `"banned"`, or `"unbanned"`   |
| actionOn  | User object of the user who was kicked/banned/unbanned     |
| actionBy  | User object of the user who performed the action           |
| actionFor | Group object of the group where the action occurred        |

| Parameter | Type   | Required | Description                                    |

| --------- | ------ | -------- | ---------------------------------------------- |

### unbanGroupMember()

```javascript
CometChat.unbanGroupMember(GUID: string, UID: string): Promise<Object>
```

| Parameter | Type   | Required | Description                                    |
| --------- | ------ | -------- | ---------------------------------------------- |
| GUID      | string | Yes      | The unique identifier of the group             |
| UID       | string | Yes      | The unique identifier of the user to unban     |

### BannedMembersRequestBuilder

```javascript
new CometChat.BannedMembersRequestBuilder(GUID: string)
  .setLimit(limit: number)
  .setSearchKeyword(searchKeyword: string)
  .build(): BannedMembersRequest
```

| Method            | Parameter     | Type   | Description                                    |
| ----------------- | ------------- | ------ | ---------------------------------------------- |
| constructor       | GUID          | string | The unique identifier of the group             |
| setLimit          | limit         | number | Maximum number of results per page (max 100)   |
| setSearchKeyword  | searchKeyword | string | Search string to filter banned members         |

### Return Values

All kick/ban/unban methods return a `Promise<Object>` that resolves to a success response object.

The `fetchNext()` method on `BannedMembersRequest` returns a `Promise<GroupMember[]>` containing the list of banned members.

### Errors

| Error Code              | Description                                    | Resolution                                      |
| ----------------------- | ---------------------------------------------- | ----------------------------------------------- |
| ERR_USER_NOT_LOGGED_IN  | No user is currently logged in                 | Ensure user is logged in before performing action |
| ERR_INVALID_GUID        | The provided GUID is invalid                   | Verify the GUID format and existence            |
| ERR_GROUP_NOT_FOUND     | Group with the specified GUID not found        | Check if the group exists                       |
| ERR_NOT_A_MEMBER        | User is not a member of the group              | User must be a member to perform moderation     |
| ERR_PERMISSION_DENIED   | User doesn't have permission                   | Only admins and moderators can kick/ban/unban   |
| ERR_USER_NOT_FOUND      | The specified UID doesn't exist                | Verify the UID exists in your CometChat app     |
| ERR_ALREADY_BANNED      | User is already banned from the group          | Check ban status before banning                 |
| ERR_NOT_BANNED          | User is not banned (for unban operation)       | Verify user is banned before unbanning          |

## Edge Cases & Pitfalls

<Warning>
**Kick vs Ban Behavior**: A kicked user can rejoin the group immediately, while a banned user cannot rejoin until they are explicitly unbanned. Choose the appropriate action based on your moderation needs. For temporary removals, use kick. For permanent or serious violations, use ban.
</Warning>

<Warning>
**Permission Hierarchy**: Admins can kick/ban moderators and participants, but moderators can only kick/ban participants. Neither can kick/ban the group owner. Always verify the target user's scope before attempting moderation actions to avoid permission errors.
</Warning>

<Warning>
**Duplicate Listener IDs**: If you register a GroupListener with the same `listenerId` as an existing listener, the new listener will replace the old one. Use unique listener IDs (e.g., include timestamps or UUIDs) to avoid accidentally overwriting listeners, especially in applications with multiple components listening for group events.
</Warning>

<Warning>
**Unbanning Non-Banned Users**: Attempting to unban a user who is not banned will result in an error. Always check the banned members list or handle the error gracefully when implementing unban functionality.
</Warning>

## Performance & Scale Notes

- The `BannedMembersRequestBuilder` supports pagination with a maximum of 100 results per page. Use `fetchNext()` repeatedly to retrieve all banned members.
- For groups with many banned members, implement lazy loading in your UI to avoid fetching all banned members at once.
- Kick, ban, and unban operations are individual API calls. If you need to moderate multiple users, consider batching the operations with appropriate error handling for partial failures.
- Real-time events are delivered to all group members, so large groups may generate significant event traffic during moderation activities.

## Security Notes

<Note>
**Auth Key** is acceptable for POC/development only. For production, use **Auth Token** created server-side via the CometChat REST API.
</Note>

<Warning>
**Server-Side Moderation**: For production applications, consider implementing server-side moderation workflows. This allows you to log moderation actions, implement appeals processes, and enforce business rules before the CometChat API is called.
</Warning>

<Warning>
**Audit Trail**: CometChat stores moderation actions as Action messages in the group history. However, for compliance requirements, consider maintaining your own audit log of moderation actions with additional context like reason for action.
</Warning>
