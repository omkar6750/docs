The **CometChat Calls SDK for Android** adds real‑time 1:1 & group voice/video calling to your app with a lightweight, event‑driven API. Use it to build a fully custom experience or pair it with the [Android UI Kit](/ui-kit/android/overview) to accelerate delivery.

***

## **Why Choose the Calls SDK?**

* **Custom UI Freedom** – Bring your own UI (Views / Compose) – SDK only handles logic & media.
* **1:1 & Group Sessions** – Scale from quick peer calls to multi‑participant conferences.
* **Reliable Signaling** – Real‑time callbacks for lifecycle & participant changes.
* **Media Controls** – Mute, pause video, switch camera, audio routing.
* **Production Ready** – Adaptive quality & resilient connections.
* **Works With Core Chat** – Leverage user profiles, presence, moderation & messaging.

***

## **Capabilities At a Glance**

| Category | Features |
|----------|----------|
| Call Types | 1:1 Voice, 1:1 Video, Group / Conference |
| Controls | Mute / Unmute, Pause / Resume Video, Switch Camera, Audio Mode |
| Events | User join/leave, end, recording toggle, audio device changes, errors |
| Flows | Direct Calling (session ID), Default Calling (invite / accept / reject) |
| Integration | Works with Core Chat SDK & optional UI Kit |

## **Getting Started (Quick Path)**

1. Create a CometChat app & copy App ID / Region / Auth Key.  
2. Add repository + dependency: `com.cometchat:calls-sdk-android:5.0.0-alpha.2`.  
3. Initialize Core Chat SDK, then Calls SDK.  
4. Login a user.  
5. Generate a call token (Direct) or obtain session from Call object (Default).  
6. Build `CallSettings` & invoke `CometChatCalls.startSession()`.  
7. Manage events & controls (mute, camera, end).  

Full walkthrough: [Integration Guide](/calls-sdk/android/integration)

## **Best Practices**

* Request CAMERA / RECORD_AUDIO only right before first call.
* Provide clear states: connecting → in‑call → ending.
* Re‑attach video surfaces after configuration changes.
* Release resources by calling `CometChatCalls.endSession()` on exit.
* Log callbacks in development to validate ordering.
