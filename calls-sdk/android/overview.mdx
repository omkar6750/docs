The **CometChat Calls SDK for Android** lets you add high‑quality, low‑latency 1:1 and group voice & video calling to your Android application. It provides a flexible, event‑driven API (no opinionated UI) so you can:

* Build a fully custom calling interface, or
* Combine it with the [CometChat UI Kit](/ui-kit/android/overview) for a faster time‑to‑market.

Designed for reliability, scalability, and real‑time performance, the Calls SDK works alongside the [Core Chat SDK](/sdk/android/overview) for user management, presence, and messaging scenarios.

***

## **Why Choose CometChat Calls SDK?**

* **Flexible UI Ownership** – You control every pixel; bring your own views or pair with our UI Kit call components.
* **1:1 & Group / Conference Calling** – Support private calls or multi‑participant sessions.
* **Real-Time Signaling & Call Events** – Rich lifecycle callbacks (initiated, ringing, joined, left, ended, errors).
* **Device & Media Controls** – Mute/unmute, camera toggle, switch camera, audio routing (where supported).
* **Scalable & Resilient** – Built for production workloads with adaptive media quality.
* **Interoperable Ecosystem** – Works seamlessly with CometChat users, presence, moderation & messaging.
* **Separation of Concerns** – Decouple messaging UI from calling logic for cleaner architecture.

## **High-Level Architecture**

The Calls SDK sits on top of CometChat's signaling layer and media infrastructure:

1. **Authentication** via Core Chat SDK (App ID, Region, Auth Key / Token)
2. **Session Establishment** (Direct Calling or Default Calling flow)
3. **Media Negotiation** (WebRTC under the hood – abstracted APIs)
4. **Event Propagation** (Listeners for call state, participants, media changes, errors)
5. **UI Rendering** (Your custom SurfaceView/TextureView / Compose components)

## **Getting Started**

Follow these steps to place your first call:

1. **Get your application keys** – Create an app & note App ID, Region & Auth Key. See [Core Overview](/sdk/android/overview#get-your-application-keys).
2. **Add the Calls SDK dependency** – See [Setup & Integration](/sdk/android/setup-calling) (repository + Gradle dependency).
3. **Initialize CometChatCalls** – Configure `CallAppSettings` (App ID, Region). See [Setup](/sdk/android/setup-calling#initialize-cometchatcalls).
4. **Login a user** – Use the Core Chat SDK login (required before calling).
5. **Choose a flow** – [Direct Calling](/sdk/android/direct-calling) (session id based) or [Default Calling](/sdk/android/default-calling) (invite / accept / reject flow).
6. **Render media** – Attach provided media streams to your video surfaces.
7. **Add call controls** – Implement mute, end call, camera toggle, participant list.

## **Integration & Customization Tips**

| Area | Guidance |
|------|----------|
| Threading | Run UI updates on the main thread; heavy processing offload to background threads. |
| Permissions | Request CAMERA + RECORD_AUDIO before initiating/joining video/voice calls. |
| Lifecycle | Re-attach video surfaces on activity/fragment recreation. |
| Error Handling | Listen for failure callbacks & show actionable UI (retry, escalate). |
| Performance | Release media tracks promptly after call end to free resources. |

## **Best Practices**

* Defer requesting camera/mic permissions until just before first use (improves accept rate).
* Provide clear visual states: connecting, ringing, in-call, reconnecting.
* Gracefully handle network changes (show a reconnecting toast / banner).
* Log call events during development to validate lifecycle ordering.
* Test with varied device orientations & background / foreground transitions.
